# 広告運用 ① ② レポート自動取得 — API調査メモ

① 広告レポートの自動取得、② 指定フォルダへの保存 を実現するためのAPI・手段のメモです。

---

## Amazon

| 項目 | 内容 |
|------|------|
| **API名** | Amazon Advertising API |
| **取得できるレポート例** | スポンサープロダクト 検索ワードレポート、キャンペーンレポート など |
| **認証** | LWA（Login with Amazon）OAuth、リフレッシュトークン。広告管理画面から開発者登録・アプリ作成が必要。 |
| **GAS連携** | UrlFetchApp で REST 呼び出し。トークン取得・更新を GAS 側で実装するか、Script Properties にリフレッシュトークンを保存して利用。 |
| **参考** | [Amazon Advertising API ドキュメント](https://advertising.amazon.com/API/docs)、テンプレート（TēPs 等）で「広告レポート→スプレッドシート」の事例あり。 |

※ セラーセントラル（SP-API）は在庫・注文向け。広告レポートは **Advertising API** を使用する。

---

## 楽天

| 項目 | 内容 |
|------|------|
| **API名** | 楽天広告 API（RPP 等）。商品出品で既にAPI連携済みとのこと。 |
| **取得できるレポート例** | 広告レポートCSVの取得可否は、利用中の楽天API（出品用）の権限・別途「広告API」の申請の有無に依存。 |
| **確認したいこと** | 現在の楽天APIで「広告パフォーマンスレポート」や「商品別レポート」を取得するエンドポイントがあるか。なければ楽天広告側の開発者向けドキュメントでレポート取得APIを確認。 |

---

## 実装の流れ（案）

1. **Amazon** … Advertising API のアカウント・アプリ作成と認証（LWA）を確認 → レポート作成・取得の API を特定 → GAS で「レポートリクエスト → 完了待ち → ダウンロード → 指定DriveフォルダにCSV保存」を実装。
2. **楽天** … 広告レポート取得用のAPI・権限を確認 → 同様に GAS から「取得 → 指定フォルダ保存」を実装。
3. **トリガー** … 隔週金曜 3:00 のトリガー実行前に、上記で「指定フォルダにCSVが入る」ようにする（① ② を別トリガーで前日などに実行する、または ① ② を隔週金曜のトリガー内で先に実行する など）。

次のステップでは、利用予定の Amazon / 楽天 のアカウント・API 種別を教えていただければ、より具体的な手順と関数名レベルで設計できます。
