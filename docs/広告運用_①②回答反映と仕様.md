## 楽天について（補足）

共有いただいた利用機能一覧には、**LicenseManagementAPI / R-Cabinet / カテゴリ / ジャンル・商品属性 / 商品API 2.0 / 在庫API 2.0・2.1** のみでした。**広告（RPP）のパフォーマンスレポート取得**に相当するAPIは一覧にありません。楽天広告レポートをAPIで取得するには、**広告用の別API・利用申請**の有無を楽天側に確認する必要があります。実装は **Amazon を優先**し、楽天は手動保存のまま、後日APIが使えるようになったら追加する形にします。

## 確定した仕様（① ②）

- **対象** … まず **Amazon 広告レポート** のみ（楽天は手動のまま）。
- **取得レポート** … **スポンサープロダクト広告の検索用語データ**（現在手動で保存しているCSVと同じ内容をAPIで取得）。
- **保存先** … 既存の **AMZ_FOLDER_ID** のフォルダにCSVを保存。
- **実行** … **隔週金曜 3:00** のトリガーで、**「① Amazonレポート取得＆保存 → ② 分析＆バルク生成」** の順で一括実行。  
  - 既存の `runOptimizationAndBulkOnSchedule` の**先頭**で、Amazon レポート取得＆保存の処理を呼び出す形に拡張する。
- **Amazon API** … **Amazon Advertising API** の登録・アプリ作成が未完了の場合は、**登録が完了してから** GAS 側の実装（レポートリクエスト → ポーリング → ダウンロード → Drive 保存）に進む。

## Amazon Ads API オンボーディング（3ステップ）

画面の説明どおり、**クライアントアプリの構築**は次の3ステップです。

| ステップ | 内容 | 備考 |
|----------|------|------|
| **1. LwA アプリを作成** | Login with Amazon でクライアントアプリを作成する。 | **無料・承認不要**。ここから開始。 |
| **2. API アクセスを申請** | 利用目的などの申請フォームと、API ライセンス契約・データ保護ポリシーへの同意。 | **承認に最大1営業日**かかることがある。 |
| **3. LwA アプリに API アクセスを割り当て** | 承認後、作成した LwA アプリに API アクセス権を紐付ける。 | 紐付け後に API リクエストが可能になる。 |

**最初にやること** … 左サイドバーの **「Amazonログインアプリケーションを作成する」**（または英語で "Create a Login with Amazon application"）をクリックし、**ステップ1** を進めてください。

### ステップ1 の詳細（developer.amazon.com にログイン済みの場合）

1. **「コンソール」**（Developer Console）をクリックする。
2. コンソールのメニューから **「Login with Amazon」** を選ぶ。
3. **「Create a New Security Profile」**（新しいセキュリティプロファイルを作成）をクリックする。
4. フォームに次を入力する：
   - **Security Profile Name** … 任意の名前（例：`AmazonAdsAPI_Octas`）。
   - **Security Profile Description** … 任意の説明（例：広告レポート取得用）。
   - **Consent Privacy Notice URL** … 自社のプライバシーポリシーURL。なければ `https://example.com/privacy` など任意のURLで可（自社データのみ利用する場合は必須ではない）。
5. **「Save」** をクリックして保存する。
6. 保存後、**Login with Amazon の一覧**に戻る。作成したプロファイルの行で **「Show Client ID and Client Secret」** をクリックし、**Client ID** と **Client Secret** をメモする（GAS の Script Properties に後で登録する）。
7. ステップ1はここまで。次は **ステップ2「API アクセスを申請」** に進む。

## Amazon Advertising API が英語で不安な場合

- 公式ドキュメントは英語ですが、**実装時は必要な手順・パラメータを日本語でまとめたメモやコメント**をこちらで用意できます。
- 登録の流れは上記オンボーディングの3ステップに沿って進め、**LWA の Client ID / Client Secret / リフレッシュトークン** が取得できたら、GAS の Script Properties に保存し、**レポート作成 API → 完了待ち → ダウンロード → 指定フォルダに保存** の実装に進みます。  
- 登録やトークン取得で分からないステップがあれば、**画面の名前や選択肢をそのまま**教えていただければ、日本語で「ここでは何を選ぶか」を整理します。

## 次のアクション

1. **Amazon Advertising API** … 開発者登録・アプリ作成・トークン取得を進める。英語が不安な箇所は、画面名や選択肢を共有いただければ日本語で整理します。
2. **登録完了後** … GAS 側で「Amazon レポート取得 → AMZ_FOLDER_ID に保存」する関数を実装し、`runOptimizationAndBulkOnSchedule` の先頭でそれを呼ぶように変更する。
3. **楽天** … 当面は手動でフォルダに保存。広告レポート用APIの利用可否を楽天に確認できたら、同様の「取得→保存」を追加する。

登録がまだの場合は「これから登録する」、すでに登録済みの場合は「登録済み・トークンあり」など、現状を教えていただければ、次の具体的な手順（GAS の関数名や設定項目）を書きます。
