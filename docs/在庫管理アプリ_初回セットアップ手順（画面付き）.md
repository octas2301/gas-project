# 在庫管理アプリ — 初回セットアップ手順（どこでどうするか）

## なぜ今 GAS が空に見えるか

- GAS エディタに **Code.gs（myFunction だけ）** しかない状態 = **まだローカルのコードを push していない**ためです。
- 次の **Step 0** で `clasp push` を実行すると、**Init.gs / SheetService.gs / index.html** などがアップロードされ、左の「ファイル」一覧に表示されます。

---

## Step 0：コードを GAS に反映する（必須）

**場所：** パソコンの **PowerShell（ターミナル）**

1. 次のコマンドを実行する。
   ```powershell
   cd C:\Users\takuy\Desktop\gas-project\inventory-app
   clasp push
   ```
2. 「Pushed X files.» のように表示されれば成功。
3. **GAS エディタのページを更新（F5 または再読み込み）** する。
4. 左側の「ファイル」に **Code.gs, Init.gs, SheetService.gs, InventoryLogic.gs, MailJob.gs, index.html** などが表示されていれば OK。

ここまでで「アップスクリプトは空」ではなくなります。

3. **「スクリプト プロパティを追加」** をクリック。
4. プロパティ: `SPREADSHEET_ID`  
   値: `1_kqBcQTcL0Q5--KsAdxMYbPMsaToLEXxOMuDw_z-kwE`  
   を入力して保存。

（未設定でもこの ID がデフォルトで使われるようになっていますが、設定しておくと安心です。）

## Step 2：Init.initializeSpreadsheet を 1 回実行（シート・初期データ作成）

**場所：** GAS エディタ（ブラウザ）

1. 左の「ファイル」で **Init.gs** をクリックして開く。
2. 上部の **「関数を選択」** ドロップダウン（初期は「関数なし」）をクリック。
3. 一覧から **`initializeSpreadsheet`** を選ぶ。
4. **「実行」** ボタン（▶）をクリック。
5. 初回は **権限の承認** が求められるので、「権限を確認」→  Google アカウントを選ぶ → 「許可」。
6. 承認後、もう一度 **「実行」** をクリック。
7. 実行が完了すると、指定したスプレッドシートに「アクセス許可」「設定」「日報宛先」「商品マスタ」「場所マスタ」「理由マスタ」「担当者マスタ」などのシートが作成され、マスタに初期データが入ります。

   `https://docs.google.com/spreadsheets/d/1_kqBcQTcL0Q5--KsAdxMYbPMsaToLEXxOMuDw_z-kwE/edit`  
   を開く。

**場所：** 同じ **Google スプレッドシート**

## Step 5：Web アプリのデプロイ

**場所：** **GAS エディタ**（ブラウザ）

1. 右上の **「デプロイ」** ボタンをクリック。
2. **「新しいデプロイ」** をクリック。
3. **「種類を選択」** の右の歯車アイコンをクリック → **「ウェブアプリ」** を選ぶ。
4. **説明**（任意）を入力。
5. **「次のユーザーとして実行」**: **自分**。
6. **「アクセスできるユーザー」**: **全員**（※匿名は不可。ログインした人のみ。doGet 側で「アクセス許可」シートをチェックします）。
7. **「デプロイ」** をクリック。
8. 表示された **ウェブアプリの URL** をコピーし、スマホやブラウザで開くと在庫管理アプリの画面が表示されます。

## まとめ（どこで何をするか）

| やること | どこで | 何をする |
|----------|--------|----------|
| コードを反映 | PowerShell | `cd inventory-app` → `clasp push` → GAS を再読み込み |
| スプレッドシート ID | GAS エディタ → プロジェクトの設定 | スクリプト プロパティに `SPREADSHEET_ID` を追加 |
| シート・初期データ | GAS エディタ | Init.gs を開く → 関数で `initializeSpreadsheet` 選択 → 実行 |
| アクセス許可 | スプレッドシート | 「アクセス許可」シートの 2 行目以降にメールアドレスを入力 |
| 日報宛先 | スプレッドシート | 「日報宛先」シートにメールアドレスを 1 行 1 件で入力 |
| Web アプリ公開 | GAS エディタ | デプロイ → 新しいデプロイ → ウェブアプリで設定 → デプロイ |

**今「アップスクリプトは空」と見えているのは、Step 0 の `clasp push` をまだ実行していないためです。まず Step 0 を実行し、GAS を再読み込みしてから Step 1 以降に進んでください。**

## トラブルシュート：読み込み中のまま進まないとき

- **調査用ログの出し方**  
  Web アプリの URL の末尾に **`?debug=1`** を付けて開く（例: `.../exec?debug=1`）。
- **ログの表示場所**
  - **画面上**：「読み込み中…」の**すぐ下**に、**黄色い枠**（「調査用ログ」）が表示されます。ここにステップ番号やエラーが出ます。
  - **ブラウザのコンソール**：キーボードで **F12** を押す → **コンソール** タブを開く。同じメッセージがこちらにも出ます。
- ログが一切出ない場合は、**clasp push とデプロイの「新バージョン」が反映されているか**を確認してください。
- **通常運用時**は調査用ログは表示されません。表示するときは URL に **`?log=1`** または **`?debug=1`** を付けて開いてください。

### GAS側（サーバー）のログ

- Web アプリの URL を開くたびに、**同じスプレッドシート**に **「デバッグログ」** シートが自動作成され、doGet の処理経過が追記されます。
- スプレッドシートを開き、一番下のタブ **「デバッグログ」** を開いて確認してください。
- 列：**日時** / **メッセージ**。`doGet start` → `loginInfo=...` → `access ok` → `safeView=...` → `template created` → `evaluate done` と出ていればサーバー側は正常です。`ERR: ...` が出ていればその行で例外が発生しています。

## トラブルシュート：Console に SyntaxError / 白画面がまだ出る場合

**「missing ) after argument list」や「Cannot set properties of null」が userCodeAppPanel で出ている**ときは、**修正したコードが Web アプリにまだ反映されていない**可能性が高いです。次の 3 点を順に確認してください。

### 1. clasp push ができているか

- **PowerShell** で `inventory-app` フォルダに移動し、**`clasp push`** を実行する。
- 「Pushed X files.» と出ていれば、GAS エディタ側のコードは更新されています。
- **GAS エディタのページを一度 F5 で再読み込み**し、index.html などが最新になっているか確認する。

### 2. Web アプリの「バージョン」を新しくしているか（重要）

- **push しただけでは、すでに公開している Web アプリの URL の内容は変わりません。**
- GAS エディタで **「デプロイ」** → **「デプロイを管理」** を開く。
- 一覧にある **ウェブアプリのデプロイ** の右側 **「編集」（ペンアイコン）** をクリック。
- **「バージョン」** を **「新バージョン」** に変更する。
- **「デプロイ」**（または「バージョンを保存」）を押して保存する。
- これで、同じ URL でも「最新のコード」が使われるようになります。

### 3. ブラウザのキャッシュを無視して開き直す

- Web アプリの URL を開くとき、**Ctrl + Shift + R**（Mac は **Cmd + Shift + R**）で **キャッシュを無視して再読み込み**する。
- または、F12 → **Network** タブで **「Disable cache」** にチェックを入れた状態で、もう一度 URL を開く。

### 反映されているかどうかの確認

- アプリが表示された状態で、画面上で **右クリック** → **「ページのソースを表示」**（または **Ctrl+U**）を開く。
- **`<body`** の近くに **`data-view="shipping"`** や **`data-view="move"`** のような **data-view** が含まれていれば、修正版が反映されています。
- また、**`currentView = (document.body.getAttribute('data-view')`** という文字列が &lt;script&gt; 内にあれば、新しいスクリプトが読み込まれています。
- これらが**ない**場合は、まだ古いバージョンが表示されているので、上記 1〜3 をもう一度確認してください。
