# ④ 候補の提案・⑤ バルク生成の自動化 — 実装前の確認

> **実装済み**（ご選択: ④A・⑤A）  
> - ④ 推奨だけレ点ON … 楽天「入札強化候補」の好調行もレ点ONに変更済み。  
> - ⑤ メニュー「1-2. 分析＆バルク生成を実行」を追加し、分析→バルクを連続実行。

以下の案で実装してよいか、番号で選ぶか・文言で修正希望を教えてください。

---

## ④ 候補を提案してほしい

### 現状
- 分析実行後、**除外候補・入札調整候補の「判定」列（1列目）は、すべてレ点ON（true）**で出力されています。
- 楽天の「入札強化候補」だけ、好調な行はレ点OFF（false）で出しています。
- そのため、**どれにレ点を付けるかは人間がシートを見て判断し、不要な行のレ点を外す**運用になっています。

### 提案する動き（どれにしますか？）

**A) 推奨だけレ点ON（おすすめ）**  
- 分析結果の「判定結果」列（除外 / 昇格 / 調整 など）に基づき、**「除外」「昇格」「調整」の行だけレ点ON**、それ以外は**レ点OFF**で出力する。  
- 人間は「推奨候補」が最初からレ点で示され、必要なら手でレ点を追加・解除できる。

**B) 従来どおり全行レ点ON ＋「推奨レ点を付ける」ボタン**  
- 分析後は**今まで通り全行レ点ON**のまま。  
- メニューに **「推奨レ点を付ける」** を追加。実行すると、**判定結果が「除外」「昇格」「調整」の行だけレ点ON**にし、他はOFFにする。  
- 「いったん全部OFFにしてから推奨だけ付ける」を1クリックでできる。

**C) 推奨行のハイライトのみ**  
- レ点の初期値は**変更しない**（現状どおり）。  
- **推奨したい行（除外・昇格・調整）の背景色を薄い色（例：黄）**で塗り、どの行が推奨か視覚で分かるようにする。  
- レ点を付けるかどうかは従来どおり人間が判断。

**D) その他**  
- 上記以外の動きを希望する場合は、具体的に書いてください。

## ⑤ バルク生成の自動化

### 現状
- **「1. 分析を実行する」** と **「2. バルク生成＆ドライブ保存」** は別メニューで、**それぞれ手動で実行**しています。

### 提案する動き（どれにしますか？ 複数可）

**A) メニュー統合：「分析＆バルク生成」を追加**  
- メニューに **「1-2. 分析＆バルク生成を実行」**（または同じ意味の名前）を1つ追加する。  
- 実行すると **「1. 分析を実行する」→ 続けて「2. バルク生成＆ドライブ保存」** の順に実行する。  
- いまの「1. 分析を実行する」「2. バルク生成～」も残し、必要なときだけ個別実行できるようにする。

**B) 分析完了後に確認ダイアログ**  
- **「1. 分析を実行する」** の最後で **「分析が完了しました。バルク生成も実行しますか？（はい/いいえ）」** を表示する。  
- 「はい」ならそのまま **「2. バルク生成～」** を実行する。「いいえ」なら分析のみで終了。

**C) 時間トリガーで定期実行**  
- 例：**毎週○曜日の ○時** に、「1. 分析を実行する」→「2. バルク生成～」を自動で順に実行する。  
- この場合、**実行時点で指定フォルダに Amazon・楽天のCSVが入っている必要**があります（CSVの準備は手動 or 将来のAPI連携）。

**D) A と B の両方**  
- 「分析＆バルク生成」メニューを追加しつつ、**「1. 分析を実行する」** の最後には「バルク生成も実行しますか？」の確認を出す。  
- どちらからでも「分析→バルク」を続けて実行できるようにする。

**E) その他**  
- 上記以外（例：実行曜日・時刻の希望、トリガーは使わない など）があれば書いてください。
