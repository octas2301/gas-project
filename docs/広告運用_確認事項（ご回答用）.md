## 4. 運用フローの確認

現在のコードで想定している流れは以下です。  
**実際の運用と違う点があれば教えてください。**
　現状は相違なし

1. **2週間に1回など**、Amazon・楽天の広告レポートCSVをダウンロードする。
2. それらを **上記の Amazon用フォルダ・楽天用フォルダ** に保存する。
3. 広告運用スプシで **「🚀 広告運用メニュー」→「1. 分析を実行する」** を実行する。  
   → 元データシートにCSVが取り込まれ、除外候補・入札候補シートが更新される。
4. 除外・入札したい行に **レ点（チェック）** を付ける。
5. **「2. バルク生成＆ドライブ保存」** を実行する。  
   → 各フォルダ内の「📦生成されたバルクファイル」にCSVが保存される。
6. そのCSVを **各モールの管理画面からアップロード** する（手動 or 既存の仕組み）。
