# Amazon 出品 要件整理

**位置づけ**: 優先順位3「出品の自動化」のうち、Amazon は**要件のみ整理**。実装は後日（選択肢C）。楽天・Yahoo! は既存 gas-project をそのまま継続運用する。

参照: [FLOW_AND_PRIORITY.md](FLOW_AND_PRIORITY.md) §8「Amazon の要件整理」

---

## 1. フロー

- **入力**: マスタ（▼商品マスタ(人間作業用)）／AI情報取得data（既存マスタ連携）
- **中間**: 商品出品情報（最小必須項目を揃える）
- **出力**: **Amazon 出品**（リスティング登録。必要に応じ広告は別途要件化）

楽天・Yahoo と同様、商品出品情報を一元化したうえでモール別に出力する。

---

## 2. 共通必須項目（他モールと共通）

§4「既存マスタ連携による最小必須項目の対応」と同様。マスタ／AI情報取得data の既存列で賄う。

| 項目 | マスタの列（想定） | 備考 |
|------|---------------------|------|
| 商品識別子 | 親SKU, 子SKU, ASINコード, JANコード 等 | 突合は ASIN/JAN/商品名 |
| 商品名 | 商品名 |  |
| 価格 | **販売価格amazon** | モール別価格列（Phase 2 確定） |
| 在庫数 | 在庫数（共通1列） |  |
| 画像 | ▼マスタ(参考情報(画像URL)) 等 |  |
| 説明・キャッチ | ▼マスタ(商品説明)、確定値列 等 | 人間編集後の確定値で出品 |

参考資料 CSV（`参考資料：出品自動化スプシデータ` の「必要sheetCSV形式違い」）に、対象ASIN・商品名・販売価格・在庫数・画像URL 等の列があり、Amazon 用項目も含まれる。実装時のマッピング基準とする。上記共通必須項目は [FLOW_AND_PRIORITY.md](FLOW_AND_PRIORITY.md) §4 の対応表と一致する。

---

## 3. Amazon 固有項目・マスタ/CSV列対応と必須/任意

リスティング登録に必要な項目。参考資料「必要sheetCSV形式違い」**行4がヘッダー**。以下は実装時のマッピング基準とする。

### 3.1 マスタ／参考資料CSV列との対応（実装で使用する列）

| 参考資料CSV行4の列名（例） | マスタの列または生成ルール | 必須/任意 |
|----------------------------|----------------------------|-----------|
| 商品管理番号 | 親SKU／子SKU から生成（例: 親SKU-子SKU末尾）。マスタに「商品管理番号」列があればその値 | 必須 |
| 販売価格 | **販売価格amazon**（Phase 2 確定） | 必須 |
| 在庫数 | 在庫数（共通1列） | 必須 |
| 商品コード(JANコード等) | マスタの 商品コード(JANコード等) または JANコード | 必須 |
| 商品コードのタイプ | EAN / JAN 等。マスタに列がなければ固定値で補完 | 必須 |
| 商品のコンディション | 新品=New 等。マスタに列がなければ固定値 | 必須 |
| 商品のコンディション説明 | 任意説明文。マスタの説明系列または空 | 任意 |
| 対象ASIN | マスタの ASINコード | 必須（既存ASINと紐づける場合） |
| 商品名 | 商品名 | 必須 |
| リードタイム(出荷までにかかる作業日数) | マスタの列または固定値（日数） | 必須 |
| 出品者SKUのメイン画像URL | ▼マスタ(参考情報(画像URL)) または 楽天メイン画像1 | 必須 |
| 出品者SKUのサブ画像URL1〜5 | 楽天サブ画像1〜8 等、マスタの画像列から対応 | 任意（1枚以上推奨） |
| フルフィルメントセンターID | FBAの場合はセンターID、自己発送の場合は空等。マスタまたは固定値 | 必須 |
| セール価格・セール開始日・セール終了日 | マスタに列があれば使用。なければ未設定 | 任意 |
| ギフト包装・ギフトメッセージ | マスタまたは固定値 | 任意 |
| 商品タックスコード・配送パターン | カテゴリに応じた設定。実装時にマッピング確定 | 任意 |
| 電池・危険物・重量・体積等 | 該当商品のみ。参考資料の列名のままマスタと対応を定義 | 任意 |
| メーカー希望価格 | マスタに列があれば使用 | 任意 |

マスタに上記の列が存在しない場合は、列追加または固定値・生成ルールで補う。既存の syncAiDataToMaster・楽天CSV・Yahoo が参照している列は変更しない。

### 3.2 必須/任意の整理（リスティング登録の観点）

- **必須**: 商品管理番号（SKU）・商品コード（JAN/EAN）・商品コードのタイプ・コンディション・価格・在庫数・商品名・画像（メイン1枚以上）・リードタイム・フルフィルメントの指定。既存ASINと紐づける場合は対象ASIN。
- **任意**: セール価格・ギフト包装・コンディション説明・サブ画像2枚目以降・商品タックスコード・配送パターン・電池・危険物・重量・体積・メーカー希望価格 等。カテゴリや商品タイプによりプラットフォーム側で必須になる項目は実装時に追加する。

参考資料に含まれる Amazon 関連列（Amz:感熱性, Amz:一人分数量, ★Amazon検索KW(150字) 等）は、実装時にマッピングと必須/任意を確定する。

---

## 4. 広告

リスティング登録後に、スポンサード商品等の範囲は**別途要件化**する。本ドキュメントでは「出品（リスティング）の必須項目・フロー」までを整理し、実装は後日とする。

---

## 5. 実装時の参照・前提・制約

- **マスタ・列名**: [MASTER_LINKAGE_TASKS.md](MASTER_LINKAGE_TASKS.md) Phase 1〜2 実施結果。価格列は「販売価格amazon」。
- **参考資料**: `参考資料：出品自動化スプシデータ` 内の「必要sheetCSV形式違い」で現状の列を確認する（行4がヘッダー）。
- **既存連携**: 既存の syncAiDataToMaster・楽天CSV出力・Yahoo の参照を**変更しない**。Amazon 用の出力または API 連携は**追加のみ**とする。
- **広告**: 本ドキュメントの対象外。リスティング登録後にスポンサード商品等は別途要件化する。
- **参考情報(画像URL)**: AI情報取得data の「参考情報(画像URL)」は、ASIN貼り付け＋Keepa取得における対象品一致判定・評価に利用する（[RESEARCH_AND_ESTIMATE.md](RESEARCH_AND_ESTIMATE.md) の画像一致・参考画像セット数対策の要件定義を参照）。
