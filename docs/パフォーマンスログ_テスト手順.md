# パフォーマンスログ テスト手順

## 1. 目的

- 各APIが「パフォーマンスログ」シートに **日時・API名・処理時間(ms)・担当者** を正しく記録することを確認する。
- 記録された処理時間から遅いAPIを把握し、改善の優先度を決める。

---

## 2. 前提

- スプレッドシートに「パフォーマンスログ」シートが存在し、ヘッダー行が  
  `日時 | API名 | 処理時間(ms) | 担当者` であること。
- アプリにアクセス許可されたユーザーでテストすること。

---

## 3. テストの進め方

### 3.1 記録の確認（全APIカバー）

次の操作を順に行い、**毎回**「パフォーマンスログ」シートを開いて **新規行が1行以上追加されていること** を確認する。

| # | 操作 | 期待されるAPI名（ログに出現） |
|---|------|------------------------------|
| 1 | アプリを開く（初回読み込み） | `apiGetLocations`, `apiGetTantouList`, `apiGetReasons`, `apiGetTantou` など |
| 2 | 担当者タブで担当者を選択・保存 | `apiSetTantou` |
| 3 | 出荷タブでJAN入力→blur | `apiLookupProduct` |
| 4 | 出荷タブで出荷登録 | `apiShipping` |
| 5 | 在庫移動タブで移動登録 | `apiMove` |
| 6 | 入荷タブで入荷登録 | `apiReceiving` |
| 7 | 棚卸タブを開く | `apiGetCountSession` |
| 8 | 棚卸タブでスタート | `apiCountStart` |
| 9 | 棚卸タブでスキャン登録 | `apiCount` |
| 10 | 棚卸タブで完了 | `apiCountComplete` |
| 11 | 差異タブを開く | `apiGetVarianceList` |
| 12 | 在庫タブを開く | `apiGetCurrentStockList` |
| 13 | 商品登録 | `apiProductAdd` |
| 14 | 棚卸確定後の修正で加算/減算を保存 | `apiCountAdjust` |

- **担当者** 列は、書き込み系API（出荷・移動・入荷・棚卸・商品登録・確定後修正・担当者設定）のときのみ値が入り、読み取り専用APIは空でよい。

### 3.2 列の妥当性

- **日時**: 操作した時刻とほぼ一致していること。
- **API名**: 上表のいずれか（または `apiGetTantou` など）であること。
- **処理時間(ms)**: 数値（ミリ秒）であること。0以上。
- **担当者**: 書き込み系のときは設定済み担当者名、読み取り系は空でよい。

### 3.3 処理時間の目安（参考）

実測例に基づく目安（環境により変動します）。

| 傾向 | API例 | 目安(ms) |
|------|-------|----------|
| 軽い | apiGetTantou | 数十〜100前後 |
| 中 | apiGetLocations, apiLookupProduct, apiGetReasons など | 数百 |
| 重い | apiGetCurrentStockList | 2500〜4300 |
| 特に重い | apiGetVarianceList | 4000〜5400 |
| 書き込み | apiShipping, apiMove, apiCount, apiReceiving など | 数百〜約3500 |

- **apiGetVarianceList** と **apiGetCurrentStockList** はデータ量に比例して遅くなるため、改善候補として優先して検討する。

---

## 4. テスト結果の記録例

次のようにチェックリストで記録するとよい。

```
[ ] 1. アプリ初回表示で apiGetLocations 等のログが付く
[ ] 2. apiSetTantou 実行でログが付き、担当者列に名前が入る
[ ] 3. apiLookupProduct が商品参照時に記録される
[ ] 4. apiShipping / apiMove / apiReceiving でログ・担当者あり
[ ] 5. 棚卸まわり apiCountStart / apiGetCountSession / apiCount / apiCountComplete が記録される
[ ] 6. apiGetVarianceList / apiGetCurrentStockList が差異・在庫表示で記録される
[ ] 7. apiProductAdd / apiCountAdjust でログ・担当者あり
[ ] 8. 日時・API名・処理時間(ms)・担当者の4列が想定どおり
```

---

## 5. 不具合時の確認ポイント

- ログが一切付かない  
  → SheetService.gs の `appendPerfLog` が存在するか、パフォーマンスログシートが作成されているか、Code.gs で `appendPerfLog` を呼んでいるかを確認。
- 特定APIだけログが出ない  
  → そのAPIの先頭で `var t0 = new Date().getTime();` を入れ、return 直前に `appendPerfLog('API名', new Date().getTime() - t0, ...);` があるか確認。
- 担当者列が常に空  
  → 書き込み系APIでは `apiGetTantou()` の戻り値を `appendPerfLog` の第3引数に渡しているか確認。

---

## 6. まとめ

- **テストの進め方**: 上記 3.1 の操作を順に実行し、毎回「パフォーマンスログ」シートに該当APIの行が追加されることを確認する。
- **記録データの使い方**: 処理時間(ms)の大きいAPIから改善の優先度を付け、特に `apiGetVarianceList` と `apiGetCurrentStockList` は要確認とする。
