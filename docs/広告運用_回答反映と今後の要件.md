## 2. 現状の実装でできること（変更不要）

- 広告運用スプシの **「1. 分析を実行する」** … 指定フォルダのCSVを読み、元データ・除外候補・入札候補シートを更新
- **「2. バルク生成＆ドライブ保存」** … レ点の付いた行からバルクCSVを生成し、フォルダ内「📦生成されたバルクファイル」に保存
- フォルダID（AMZ / RAK）は**このままでよい**（共有済みのため contact で実行可能）

## 4. 今後の進め方（提案）

1. **すぐに着手できるもの**
   - **④ 候補の提案** … 分析結果の「判定」列に基づき、「除外推奨」「入札調整推奨」の行を色分けしたり、別列に「推奨レ点」を付与するなど。既存の広告運用GAS内で拡張可能。
   - **⑤ バルク生成の半自動化** … 「1. 分析を実行する」の完了後に「2. バルク生成」を続けて実行するメニュー追加、または時間トリガーで「毎週○曜日に分析→バルク生成まで実行」など。

2. **API連携が必要なもの（調査・設計から）**
   - **① ② レポートの自動取得・保存** … Amazon Advertising API / 楽天広告API のレポート取得仕様・認証・レートリミットを確認し、GAS または別スクリプトで「指定フォルダにCSV保存」まで設計。
   - **⑥ モールへのアップロード** … 各社のバルクアップロードAPI or 管理画面の自動操作の可否を調査。実現可能なら ⑤ の出力CSVをそのままAPIに渡す流れを検討。

3. **Yahoo!**
   - Amazon・楽天の自動化（①〜⑥の範囲）が動き出したら、楽天に準じる形で Yahoo! 用の取り込み・分析・バルクを追加する。
