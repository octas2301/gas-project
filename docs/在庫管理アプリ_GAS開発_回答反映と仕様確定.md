## 2. デプロイ・アクセス権限

| 項目 | 確定内容 |
|------|----------|
| アクセス対象 | **Google でログイン必須**。かつ **「登録シート」に登録してあるアカウントのみアクセス可** とする。 |
| 実装方針 | スプレッドシート内に「**アクセス許可アカウント**」用のシート（例: `アクセス許可`）を用意し、メールアドレス（または Google アカウント識別子）を列挙。doGet 時に「現在のユーザー」がその一覧に含まれるかチェックし、含まれなければ「アクセス権がありません」画面を表示する。 |

## 3. 初回リリース（v1）に含める機能

**すべて初回で実装する。**

| 機能 | 備考 |
|------|------|
| 出荷 | 1点ずつスキャン登録 |
| 入荷 | 数量・理由・仕入単価等。平均原価の移動平均更新を含む |
| 棚卸 | JAN・数量・ロケーション。場所別に棚卸頻度が異なる運用を考慮 |
| 在庫移動 | 移動元・移動先 |
| 商品情報 | マスタ参照・編集・新規登録。箱JAN・単品JAN 対応 |
| 差異 | 理論在庫 vs 棚卸実数 |
| 現在庫 | 前回棚卸＋入荷－出荷で計算。全社表示＋階層で場所別表示 |
| 平均原価 | 入荷時の仕入単価で移動平均更新 |
| 箱JAN・単品JAN | 両方登録し単品で集計 |
| 在庫移動日報メール | 毎日 20:00（Asia/Tokyo）。複数宛先。移動 0 件の日は送らない |

## 4. 現在庫の定義

| 項目 | 確定内容 |
|------|----------|
| 前回棚卸の決め方 | **A)** 棚卸スキャンリストの **最も新しい日付** を「前回棚卸日」とみなし、その日時点の実数＋その後入荷－その後出荷で現在庫を計算する。 |
| 場所別の考慮 | 場所別に棚卸頻度が違う（例: 外注先は高め、自社倉庫は低め）。**最終棚卸日は画面・一覧に記載**する。 |
| アラート | **棚卸が一定期間（例: 3 か月）以上されていない** 場所・商品はアラート表示。**期間は任意で指定できる**ようにする。 |
| 表示単位 | **全社合計** をデフォルト表示。**階層を潜れば場所別**（オフィス・倉庫・協力会社など）ごとの現在庫を一覧表示する。 |

|------|----------|
| 送信時刻 | 毎日 **20:00**（Asia/Tokyo） |

## 7. 担当者（スキャン入力者）

| 項目 | 確定内容 |
|------|----------|
| 担当者の入れ方 | **A)** 設定画面で「現在の担当者」を 1 回入力し、以降の登録に自動で付与。**変更も選択でできる**ようにする（過去に使った担当者名を選択肢として表示する等）。 |
| ログ | **入力場所ログは残す**。誰が・どこで（どの画面で）操作したか分かるようにする（担当者名・操作種別・日時等を記録するシートまたは列を検討）。 |

## 10. 実装時に反映するポイント一覧

- **スプレッドシート ID**: `1_kqBcQTcL0Q5--KsAdxMYbPMsaToLEXxOMuDw_z-kwE` を Script Properties に登録。
- **アクセス制御**: 「アクセス許可」シート（または同等）で許可メールアドレスを管理し、doGet 時にチェック。
- **現在庫**: 最終棚卸日を表示し、棚卸が N か月以上ない場合はアラート（N は設定可能）。表示は全社→階層で場所別。
- **日報メール**: 20:00、複数宛先、0 件の日は送信しない。
- **担当者**: 設定で 1 回入力＋選択で変更。入力場所ログを残す。
- **バーコード**: カメラ読み取りを v1 に含める。
- **UI**: 動き・アイコン・色でメニューが分かりやすく、かっこよい見た目にする。
