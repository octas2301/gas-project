# 在庫管理アプリ — 現在庫と差異の計算ロジック

## 用語

| 用語 | 意味 |
|------|------|
| **前回棚卸日** | 棚卸スキャンリストに記録がある「最も新しい日付」（日のみ。時刻は見ない） |
| **理論在庫** | 画面上の「現在庫」と同じ計算結果（前回棚卸実数 ＋ その翌日以降の入荷 － その翌日以降の出荷） |
| **今月の棚卸実数** | 棚卸スキャンリストの「年月」が今月の行の、その JAN の数量合計 |

---

## 1. 現在庫（理論在庫）

**計算式（修正後）**

```
現在庫 ＝ 前回棚卸日の棚卸実数 ＋ 前回棚卸日の「翌日」以降の入荷 － 前回棚卸日の「翌日」以降の出荷
```

- **前回棚卸日の棚卸実数**  
  前回棚卸日（日付のみ）に、棚卸スキャンリストに登録されたその JAN の数量合計。
- **翌日以降の入荷**  
  前回棚卸日の翌日 00:00 以降に登録された入荷リストの数量合計。
- **翌日以降の出荷**  
  前回棚卸日の翌日 00:00 以降に登録された出荷リストの数量合計。

**「翌日以降」にする理由**  
前回棚卸日と同じ日の入荷・出荷まで含めると、  
「その日の棚卸実数」に、すでに同じ日の入出荷が反映されているのに、さらに足し引きして二重になり、現在庫が実態と合わなくなります。  
そのため、入荷・出荷は「前回棚卸日の翌日以降」だけを使います。

**ご提示のテストデータでの結果**

- 前回棚卸日: 2026/02/26（棚卸スキャンリストに 1 件あるため）
- 前回棚卸日の棚卸実数: **13**
- 翌日（2026/02/27）以降の入荷: **0**（入荷 25 は 02/26 のため含めない）
- 翌日以降の出荷: **0**（出荷 3 は 02/26 のため含めない）

→ **現在庫 ＝ 13 ＋ 0 － 0 ＝ 13**

---

## 2. 差異（棚卸差異）

**計算式**

```
差異 ＝ 今月の棚卸実数 － 理論在庫（＝ 現在庫）
```

- **今月の棚卸実数**  
  棚卸スキャンリストの「年月」が今月（例: 2026/02）の行だけを集計した、その JAN の数量合計。
- **理論在庫**  
  上記「現在庫」と同じ値。

**ご提示のテストデータでの結果（修正後）**

- 理論在庫（現在庫）: **13**
- 今月の棚卸実数: **13**（2026/02 の棚卸 1 件）

→ **差異 ＝ 13 － 13 ＝ 0**  
（差異一覧には「0 でないもの」だけ出すため、この JAN は一覧に載らない想定）

**「前回棚卸なし＝0、入荷25、棚卸13 → 0+13-25＝-12」について**

- 想定されている式は「**前回棚卸 ＋ 今月棚卸 － 入荷**」であり、  
  現在の実装は「**理論在庫 ＝ 前回棚卸 ＋ 翌日以降入荷 － 翌日以降出荷**」「**差異 ＝ 今月棚卸 － 理論在庫**」です。
- 上記のとおり「翌日以降」にそろえた結果、  
  テストデータでは 現在庫＝13・差異＝0 となり、  
  「手元の実数（棚卸 13）＝ 理論」という状態になります。
- 「0+13-25＝-12」のような差異にしたい場合は、  
  差異の定義（何を「理論」とするか・何と何の差を「差異」とするか）を別途決める必要があります。その場合は仕様として整理したうえで計算式を変更できます。

---

## 3. 修正前のロジックで起きていたこと（参考）

**現在庫**

- 修正前は「**前回棚卸日 00:00 以降**」の入荷・出荷を加減していました。
- 前回棚卸日が 2026/02/26 の場合、その日の入荷 25・出荷 3 も含まれるため、  
  **現在庫 ＝ 13 ＋ 25 － 3 ＝ 35** になっていました。
- 実際の手元は棚卸で 13 なので、「現在庫＝35」は実態と合いませんでした。

**差異**

- 理論在庫（現在庫）が 35、今月の棚卸が 13 だと、  
  **差異 ＝ 13 － 35 ＝ －22** と表示されていました。
- 現在庫の定義を「翌日以降」に直すと、理論＝13・棚卸＝13 となり、差異＝0 になります。

## 4. まとめ

| 項目 | 修正後の計算 |
|------|----------------|
| **現在庫** | 前回棚卸日の棚卸実数 ＋ **前回棚卸日の翌日以降**の入荷 － **前回棚卸日の翌日以降**の出荷 |
| **差異** | 今月の棚卸実数 － 現在庫（理論在庫） |
| **棚卸の数値を現在庫に含めるか** | 含めます。「前回棚卸日の棚卸実数」が現在庫の起点で、その後に「翌日以降の入荷・出荷」を加減します。 |

この内容でコードを修正し、現在庫が「手元の棚卸実数＋その後の入出荷」と一致するようにしています。

## 5. パターン別の計算式（一つずつ）

以下はすべて**同一の単品JAN**で、**前回棚卸日**は「棚卸スキャンリストに記録がある最も新しい日付」です。

**パターン早見表**

| パターン | 内容 | 現在庫（理論） | 差異 |
|----------|------|----------------|------|
| ① | 先月棚卸0、入荷25・出荷3→感覚22。棚卸13のみ（当日入出荷なし） | 13 | 0 ※ |
| ② | ①＋棚卸した**当日**に入荷5・出荷1 | 13 | 0 |
| ③ | 棚卸日と入出荷が**別日**（翌日に入荷5・出荷1） | 17 | －4 |
| ④ | 複数日で棚卸（2/26に10、2/27に3） | 3 | 10 |

※ ①で「不足9個」と出したい場合は理論の定義変更が必要（本文参照）。

**共通の計算式**

| 項目 | 計算式 |
|------|--------|
| **現在庫（理論）** | 前回棚卸日の棚卸実数 ＋ **翌日以降**の入荷 － **翌日以降**の出荷 |
| **差異** | 今月の棚卸実数 － 現在庫（理論） |

- **差異の読み方**: 差異が **マイナス** → 理論より少ない（不足）。**プラス** → 理論より多い（過多）。

---

### パターン①：先月棚卸0、入荷25・出荷3のあと、棚卸で13個スキャン（当日の入出荷なし）

| 項目 | 値 |
|------|-----|
| 先月の棚卸 | 0（棚卸スキャンなし＝前回棚卸日は「今月の棚卸を入れた日」になる） |
| 入荷 | 25個 |
| 出荷 | 3個 |
| その時点の在庫（感覚） | 0＋25－3＝**22個** |
| 今月の棚卸スキャン | 13個（1日だけ実施、他に入出荷なし） |

**このアプリでの計算**

- **前回棚卸日** ＝ 棚卸を登録した日（例：2/26）
- **前回棚卸日の棚卸実数** ＝ **13**
- **翌日以降の入荷** ＝ 0（入荷25は「前回棚卸日より前」または「同日」のため含めない）
- **翌日以降の出荷** ＝ 0

→ **現在庫（理論）** ＝ 13 ＋ 0 － 0 ＝ **13**

- **今月の棚卸実数** ＝ **13**
- **差異** ＝ 13 － 13 ＝ **0**

**「22個あるはずなのに13個しかスキャンされていない → 差異は9個（不足）」と出したい場合**

- 感覚的な「あるべき数」＝ 22（0＋25－3）、実数＝13 → **不足 9個**。
- 現在の実装では **理論 ＝ 前回棚卸日の棚卸実数 ＋ 翌日以降の入出荷** なので、前回棚卸日＝棚卸した日だと 理論＝13・棚卸＝13 → **差異＝0** になります。
- **差異＝9（または－9で「9個不足」）** としたい場合は、**理論**を「前回棚卸がまだない月は 0＋その月の入荷－その月の出荷」のように定義し直す仕様が必要です。その場合の例：
  - 理論 ＝ 0＋25－3＝22、棚卸＝13 → 差異＝13－22＝**－9**（9個不足）
  - または 差異表示を「理論－棚卸」にすれば 22－13＝**9**（不足数として表示）

---

### パターン②：パターン①に加えて、棚卸した「当日」に入荷5・出荷1があった場合

| 項目 | 値 |
|------|-----|
| 前回棚卸日 | 棚卸をした日（例：2/26） |
| その日の棚卸実数 | 13 |
| その日の入荷 | 5 |
| その日の出荷 | 1 |

**計算**

- **翌日以降の入荷** ＝ 0（入荷5は「当日」なので翌日以降ではない）
- **翌日以降の出荷** ＝ 0（出荷1も当日）

→ **現在庫（理論）** ＝ 13 ＋ 0 － 0 ＝ **13**  
→ **今月の棚卸実数** ＝ **13**  
→ **差異** ＝ 13 － 13 ＝ **0**

**ポイント**  
当日の入荷5・出荷1は**現在庫の計算に含めません**。前回棚卸日の「翌日」以降だけを加減するため、同じ日の入出荷は二重計上を防ぐため除外しています。  
そのため「13＋5－1＝17」とはならず、**現在庫＝13・差異＝0** のままです。

---

### パターン③：棚卸日と入出荷が「別の日」の場合（翌日に入荷5・出荷1）

| 項目 | 値 |
|------|-----|
| 前回棚卸日 | 2/26（棚卸13を登録） |
| 2/27 に入荷5・出荷1 | 翌日なのでカウント対象 |

**計算**

- 前回棚卸日の棚卸実数 ＝ **13**
- 翌日以降の入荷 ＝ **5**
- 翌日以降の出荷 ＝ **1**

→ **現在庫（理論）** ＝ 13 ＋ 5 － 1 ＝ **17**

- 今月の棚卸実数 ＝ **13**（2/26の棚卸分のみ）
- **差異** ＝ 13 － 17 ＝ **－4**（理論17に対して棚卸13 → 4個不足）

---

### パターン④：複数日に分けて棚卸した場合（今月のみ）

| 項目 | 値 |
|------|-----|
| 2/26 棚卸 | 10個 |
| 2/27 棚卸 | 3個（同一JAN） |

**計算**

- **前回棚卸日** ＝ 2/27（最も新しい棚卸日）
- **前回棚卸日の棚卸実数** ＝ **3**（2/27の分だけ。2/26の10は含めない）
- 翌日以降の入荷・出荷 ＝ 0 とすると  
→ **現在庫（理論）** ＝ 3  
- **今月の棚卸実数** ＝ 10＋3＝**13**（今月の棚卸はすべて合算）  
→ **差異** ＝ 13 － 3 ＝ **10**

**ポイント**  
「前回棚卸日の棚卸実数」は**その日付の分だけ**。過去の棚卸日（2/26）の10は含めません。今月の棚卸合計（13）との差が差異になります。

## 6. 在庫場所（ロケーション）ごとの差異について

**現状の仕様**

- **差異一覧は「単品JAN単位」のみ**です。
- 入荷・出荷・棚卸には「ロケーション」列がありますが、**差異の集計ではロケーションで分けておらず**、JANごとの合計だけを使っています。
- そのため**「オフィス1階」「倉庫」など場所ごとの差異**は、現在の画面・APIには出ていません。

**場所ごとの差異を出したい場合**

- 理論在庫・今月棚卸実数を**（JAN, ロケーション）ごと**に集計し、  
  **差異（場所別）＝ その場所の今月棚卸 － その場所の理論**  
  のような計算と、一覧表示の仕様追加が必要です。現状のコードでは未対応です。

## 7. 棚卸スタート・完了あり：期間の区分と全パターンの計算ロジック

棚卸画面に**スタート**と**完了**ボタンがある場合の想定です（現状のコードには未実装です）。

---

### 7.0 棚卸期間の分け方と「あるべき形」の確認

**ご認識のとおり、次の2つに分けます。**

| 区分 | 意味 |
|------|------|
| **棚卸期間中** | 「スタート」を押したあと、「完了」を押すまでの間。このあいだは**入出荷を加味した理論在庫**に対して、棚卸スキャンを突き合わせる。 |
| **棚卸期間外** | 棚卸をしていない、またはその回の棚卸を「完了」済みの状態。通常の現在庫・差異の表示になる。 |

**あるべき形（棚卸期間中のロジック）**

- **棚卸期間中は「入出荷を加味した理論在庫」に対して、棚卸スキャンを突合する**のがあるべき形です。
- つまり「スタート時点の理論」に、**そのあと発生した入荷を足し、出荷を引いた値**を「理論在庫」とし、それと棚卸スキャン数を比べて差異を出します。

**期間中のパターン（3つ）**

ご指摘のとおり、期間中は次の3パターンで整理できます。

| パターン | 内容 |
|----------|------|
| **① 出荷した場合** | 期間中に出荷だけした。理論 ＝ スタート時点 － 出荷。棚卸スキャンと突合。 |
| **② 入荷と出荷をした場合** | 期間中に入荷と出荷の両方した。理論 ＝ スタート時点 ＋ 入荷 － 出荷。棚卸スキャンと突合。 |
| **③ 棚卸入力して、かつ入出荷した場合** | 期間中に棚卸スキャンを入力し、かつ入荷・出荷もした。理論 ＝ スタート時点 ＋ 入荷 － 出荷。入力した棚卸数と理論を突合。 |

以下、この「入出荷を加味した理論」で突合する前提で、**期間中**と**期間外**の計算ロジックを式で書きます。

---

### 7.0.1 棚卸期間中の計算式（あるべき形：入出荷を加味した理論で突合）

**共通の定義**

| 記号 | 意味 |
|------|------|
| **T_start** | このセッションで「スタート」を押した日時 |
| **理論_スタート** | スタートを押した時点の理論在庫（JAN別で記録） |
| **入荷_期間中** | T_start より**後**に登録された入荷の、当該 JAN の数量合計（完了を押すまで増え続ける） |
| **出荷_期間中** | T_start より**後**に登録された出荷の、当該 JAN の数量合計（完了を押すまで増え続ける） |
| **棚卸_期間中** | このセッションでスキャンして登録した数量の合計（JAN別） |

**期間中の理論在庫（入出荷を加味した値）**

```
理論在庫（期間中） ＝ 理論_スタート ＋ 入荷_期間中 － 出荷_期間中
```

**期間中の差異（突合結果）**

```
差異 ＝ 棚卸_期間中 － 理論在庫（期間中）
     ＝ 棚卸_期間中 － （理論_スタート ＋ 入荷_期間中 － 出荷_期間中）
```

- 差異が **マイナス** → 理論より少ない（不足）
- 差異が **プラス** → 理論より多い（過多）

---

### 7.0.2 期間中の3パターン別の計算ロジック（すべて式で記載）

#### ① 出荷した場合

- **状況**: 棚卸期間中に、当該 JAN の**出荷だけ**が発生した。
- **入荷_期間中** ＝ 0  
- **出荷_期間中** ＝ 出荷した数量（例: 3）

**計算式**

```
理論在庫（期間中） ＝ 理論_スタート － 出荷_期間中
                   ＝ 理論_スタート － 3

差異 ＝ 棚卸_期間中 － 理論在庫（期間中）
     ＝ 棚卸_期間中 － （理論_スタート － 3）
```

**例**: 理論_スタート＝22、出荷3 → 理論在庫（期間中）＝19。棚卸スキャン13なら 差異＝13－19＝**－6**（6個不足）。

---

#### ② 入荷と出荷をした場合

- **状況**: 棚卸期間中に、当該 JAN の**入荷と出荷の両方**が発生した。
- **入荷_期間中** ＝ 入荷した数量（例: 5）  
- **出荷_期間中** ＝ 出荷した数量（例: 1）

**計算式**

```
理論在庫（期間中） ＝ 理論_スタート ＋ 入荷_期間中 － 出荷_期間中
                   ＝ 理論_スタート ＋ 5 － 1

差異 ＝ 棚卸_期間中 － 理論在庫（期間中）
     ＝ 棚卸_期間中 － （理論_スタート ＋ 5 － 1）
```

**例**: 理論_スタート＝22、入荷5・出荷1 → 理論在庫（期間中）＝26。棚卸スキャン13なら 差異＝13－26＝**－13**（13個不足）。

---

#### ③ 棚卸入力して、かつ入出荷した場合

- **状況**: 棚卸期間中に、**棚卸スキャンの入力**と**入荷・出荷**の両方が発生した。  
  「入力した棚卸数」と「入出荷を加味した理論在庫」を突き合わせる。

**計算式（②と同じ理論で、棚卸数と突合）**

```
理論在庫（期間中） ＝ 理論_スタート ＋ 入荷_期間中 － 出荷_期間中

差異 ＝ 棚卸_期間中 － 理論在庫（期間中）
     ＝ 棚卸_期間中 － （理論_スタート ＋ 入荷_期間中 － 出荷_期間中）
```

- **棚卸_期間中**: このセッションで入力したスキャン数の合計。
- **入荷_期間中・出荷_期間中**: スタート後～突合時点（または完了時点）までの入荷・出荷の合計。

**例**: 理論_スタート＝22、入荷5・出荷1、棚卸スキャン17 → 理論在庫（期間中）＝26、差異＝17－26＝**－9**。

---

### 7.0.3 棚卸期間外の計算ロジック（シンプルな形）

期間外は「入出荷を加味した理論で突合」は使いません。通常の現在庫と、完了済みセッションの差異だけを使います。

| 項目 | 計算式 |
|------|--------|
| **前回棚卸日** | 最後に「完了」を押した日 |
| **現在庫** | 前回棚卸日の棚卸実数 ＋ その翌日以降の入荷 － その翌日以降の出荷 |
| **差異（表示）** | 直近の完了セッションで確定した値（期間中に計算した「棚卸 － 理論在庫（期間中）」をそのまま表示） |

---

### 7.1 棚卸期間の定義（スタート・完了あり）

| 区分 | 定義 | 判定 |
|------|------|------|
| **棚卸期間中** | 「スタート」を押したあと、「完了」を押すまでの間。**入出荷を加味した理論在庫**に棚卸スキャンを突合する。 | 未完了の棚卸セッションが1件以上存在する状態。 |
| **棚卸期間外** | 上記以外。棚卸作業をしていない、またはその回の棚卸を完了済みの状態。 | 未完了の棚卸セッションが0件の状態。 |

- **棚卸セッション**: 1回の「スタート」～「完了」を1セッションとする。
- **未完了セッション**: スタート済みで完了していないセッション。
- **前回棚卸日**: 最後に「完了」を押した日（完了していないスキャンだけの日は含めない）。

---

### 7.2 共通の基礎式（全パターンで参照）

次の式は期間の内外を問わず、定義として共通です。

| 記号・用語 | 意味・計算式 |
|------------|----------------|
| **D_last** | 前回棚卸日（最後に「完了」を押した日付。日のみ、時刻は使わない） |
| **Q_count(D)** | 日付 D に棚卸スキャンリストに登録された、当該 JAN の数量合計 |
| **Q_in(after)** | 日時 after より**後**に登録された入荷リストの、当該 JAN の数量合計 |
| **Q_out(after)** | 日時 after より**後**に登録された出荷リストの、当該 JAN の数量合計 |
| **T_start** | 当該セッションで「スタート」を押した日時 |
| **理論_通常** | その時点での「現在庫」と同じ値（下記の「現在庫」の式で計算した値） |

**現在庫（理論_通常）の定義**

```
現在庫 ＝ Q_count(D_last) ＋ Q_in(D_lastの翌日 00:00) － Q_out(D_lastの翌日 00:00)
```

- D_last が存在しない場合（一度も完了していない場合）は、現在庫の計算方法は「スタート・完了なしモード」に依存（後述）。

---

### 7.3 棚卸期間外の計算ロジックとパターン

**期間外で使う共通ロジック**

- **前回棚卸日**: 最後に「完了」を押した日 ＝ **D_last**
- **現在庫**: 上記「現在庫」の式そのまま（D_last の棚卸実数 ＋ 翌日以降入荷 － 翌日以降出荷）
- **差異**: 直近の**完了したセッション**について計算。未完了セッションは存在しない前提。

---

#### パターン【期間外-A】スタート・完了を一度も使っていない

- **判定**: 棚卸セッションの記録が0件、またはスタート・完了機能を使わない運用。
- **前回棚卸日**: 棚卸スキャンリストに記録がある最も新しい日付（日のみ）。完了の概念がないので「最後のスキャン日」で代用。
- **現在庫**  
  ```
  現在庫 ＝ Q_count(D_last) ＋ Q_in(D_lastの翌日 00:00) － Q_out(D_lastの翌日 00:00)
  ```
- **差異**  
  ```
  今月の棚卸実数 ＝ 棚卸スキャンリストの「年月」が今月の行の当該 JAN 数量合計
  差異 ＝ 今月の棚卸実数 － 現在庫
  ```
- **入荷・出荷**: 現在庫には「D_last の翌日以降」の入荷・出荷のみ加減。同日分は含めない。

---

#### パターン【期間外-B】直近が完了済み（1回だけ完了している）

- **判定**: 未完了セッションなし。完了したセッションが1回以上ある。
- **前回棚卸日**: 最後に「完了」を押した日 ＝ **D_last**
- **現在庫**  
  ```
  現在庫 ＝ Q_count(D_last) ＋ Q_in(D_lastの翌日 00:00) － Q_out(D_lastの翌日 00:00)
  ```
- **差異**（直近完了セッションの結果を表示）  
  ```
  スタート時点理論 ＝ そのセッションのスタート押下時に記録した理論（JAN別）
  セッションの棚卸実数 ＝ そのセッションでスキャンした数量合計（JAN別）
  差異 ＝ セッションの棚卸実数 － スタート時点理論
  ```
- **入荷・出荷**: 現在庫には「D_last の翌日以降」のみ反映。スタート時点理論は過去に固定済みなので変動しない。

---

#### パターン【期間外-C】同日に2回完了している（直近は2回目の完了）

- **判定**: 同日にスタート→完了を2回行い、いずれも完了済み。未完了セッションなし。
- **前回棚卸日**: その日 ＝ **D_last**
- **現在庫**  
  ```
  Q_count(D_last) ＝ その日に登録された全スキャンの合計（1回目＋2回目）
  現在庫 ＝ Q_count(D_last) ＋ Q_in(D_lastの翌日 00:00) － Q_out(D_lastの翌日 00:00)
  ```
- **差異**（セッションごとに表示）  
  - 1回目セッション:  
    ```
    差異_1 ＝ 1回目のセッション棚卸実数 － 1回目のスタート時点理論
    ```
  - 2回目セッション:  
    ```
    差異_2 ＝ 2回目のセッション棚卸実数 － 2回目のスタート時点理論
    ```
- **入荷・出荷**: 現在庫には「D_last の翌日以降」のみ。2回目のスタート時点理論は、1回目完了直後の現在庫で固定。

---

#### パターン【期間外-D】先月に完了済みで、今月はまだスタートしていない

- **判定**: 未完了セッションなし。最後の完了は先月。
- **前回棚卸日**: 先月の完了日 ＝ **D_last**
- **現在庫**  
  ```
  現在庫 ＝ Q_count(D_last) ＋ Q_in(D_lastの翌日 00:00) － Q_out(D_lastの翌日 00:00)
  ```
- **差異**: 表示するなら「直近完了セッション（先月）」の 棚卸実数 － スタート時点理論。今月の棚卸はまだないので、今月用の差異はなし。
- **入荷・出荷**: D_last の翌日以降（先月の翌日～今月まで）の入荷・出荷が現在庫に反映される。

---

### 7.4 棚卸期間中の計算ロジックとパターン

**期間中で使う共通ロジック**

- **前回棚卸日**: 変わらない（最後に「完了」を押した日 ＝ **D_last**）。未完了のあいだは完了していないので、D_last は前の完了日のまま。
- **現在庫（画面表示）**: 期間中も「前回棚卸日」基準で計算する。スタートで新しい理論を記録しても、**現在庫の算出式は変えない**。  
  ```
  現在庫 ＝ Q_count(D_last) ＋ Q_in(D_lastの翌日 00:00) － Q_out(D_lastの翌日 00:00)
  ```
- **スタート時点理論**: このセッションで「スタート」を押した時点の現在庫を記録した値。**期間中は変動しない**（期間中の入荷・出荷はスタート時点理論に加算しない）。
- **差異**: 未完了のあいだは**未確定**。差異一覧には出さない、または「未確定」表示。**完了**を押した時点で確定。  
  ```
  差異（確定値）＝ このセッションの棚卸実数 － このセッションのスタート時点理論
  ```

---

#### パターン【期間中-1】スタート直後（スキャン0件）

- **判定**: スタートを押した直後。まだ1件もスキャンしていない。
- **前回棚卸日**: 変更なし（前回の完了日）＝ **D_last**
- **現在庫**  
  ```
  現在庫 ＝ Q_count(D_last) ＋ Q_in(D_lastの翌日 00:00) － Q_out(D_lastの翌日 00:00)
  ```
- **スタート時点理論**:  
  ```
  スタート時点理論 ＝ スタート押下時点の現在庫（上記の現在庫の式で計算した値）
  ```
- **セッションの棚卸実数**: 0  
- **差異**: 未確定（表示しない、または「—」）。

---

#### パターン【期間中-2】スタート後、同日に入荷・出荷が発生

- **判定**: スタート済み・未完了。同じ日に入荷・出荷が登録された。
- **前回棚卸日**: 変更なし ＝ **D_last**
- **現在庫**  
  ```
  現在庫 ＝ Q_count(D_last) ＋ Q_in(D_lastの翌日 00:00) － Q_out(D_lastの翌日 00:00)
  ```
  - 当日の入荷・出荷は「D_last の翌日以降」に含まれるかどうかで決まる。D_last が今日なら、当日は翌日以降ではないので**当日の入荷・出荷は現在庫に含めない**。
- **スタート時点理論**: スタート時に記録した値のまま。**当日の入荷・出荷は加算しない**。
- **セッションの棚卸実数**: このセッションでいままでにスキャンした数量の合計（未確定）。
- **差異**: 未確定。

---

#### パターン【期間中-3】スタート後、翌日以降に入荷・出荷が発生

- **判定**: スタート済み・未完了。スタート日より翌日以降に入荷・出荷が登録された。
- **前回棚卸日**: 変更なし ＝ **D_last**（最後の完了日がスタート日より前なら、D_last はその日）。
- **現在庫**  
  ```
  現在庫 ＝ Q_count(D_last) ＋ Q_in(D_lastの翌日 00:00) － Q_out(D_lastの翌日 00:00)
  ```
  - 翌日以降の入荷・出荷は、D_last の翌日以降であれば現在庫に**反映される**（期間中でも現在庫は「前回完了日」基準のため、翌日以降の入出荷で増減する）。
- **スタート時点理論**: スタート時に記録した値のまま。**翌日以降の入荷・出荷もスタート時点理論には加算しない**。
- **セッションの棚卸実数**: このセッションでいままでにスキャンした数量の合計（未確定）。
- **差異**: 未確定。確定は**完了**押下時。  
  ```
  差異（完了時）＝ セッションの棚卸実数 － スタート時点理論
  ```

---

#### パターン【期間中-4】スタート後、スキャンのみ追加（入出荷なし）

- **判定**: スタート済み・未完了。スキャンのみ追加。入荷・出荷の追加なし。
- **前回棚卸日**: 変更なし ＝ **D_last**
- **現在庫**  
  ```
  現在庫 ＝ Q_count(D_last) ＋ Q_in(D_lastの翌日 00:00) － Q_out(D_lastの翌日 00:00)
  ```
  - 未完了セッションのスキャンは**Q_count(D_last) に含めない**（D_last は「最後に完了した日」なので、未完了のスキャンはまだどの日にも確定していない）。したがって期間中は現在庫の数値はスタート前と変わらない。
- **スタート時点理論**: スタート時に記録した値のまま。
- **セッションの棚卸実数**: このセッションでいままでにスキャンした数量の合計。
- **差異**: 未確定。完了時に  
  ```
  差異 ＝ セッションの棚卸実数 － スタート時点理論
  ```

---

#### パターン【期間中-5】スタートと完了が別日（スタート日は昨日、今日スキャンして完了する前）

- **判定**: 昨日スタート、今日スキャン追加中。まだ完了していない。
- **前回棚卸日**: 変更なし（前回の完了日）＝ **D_last**
- **現在庫**  
  ```
  現在庫 ＝ Q_count(D_last) ＋ Q_in(D_lastの翌日 00:00) － Q_out(D_lastの翌日 00:00)
  ```
- **スタート時点理論**: 昨日スタート時に記録した値のまま（昨日時点の現在庫）。
- **セッションの棚卸実数**: このセッションでいままでにスキャンした数量の合計（昨日＋今日分）。
- **差異**: 未確定。完了時に  
  ```
  差異 ＝ セッションの棚卸実数 － スタート時点理論
  ```

---

### 7.5 完了時点の計算ロジック（全パターン共通）

「完了」を押した時点で、次のように確定する。

| 項目 | 計算式・処理 |
|------|----------------|
| **前回棚卸日** | この完了日 ＝ 新しい **D_last** |
| **Q_count(D_last)** | この日に登録された棚卸スキャンの数量合計（このセッションのスキャンを含む。同日に他セッションがあればその合計） |
| **現在庫** | 新しい D_last を使って再計算。  
  ```
  現在庫 ＝ Q_count(新しい D_last) ＋ Q_in(新しい D_last の翌日 00:00) － Q_out(新しい D_last の翌日 00:00)
  ```
| **差異（このセッション）** |  
  ```
  差異 ＝ このセッションの棚卸実数 － このセッションのスタート時点理論
  ```

### 7.7 用語・実装メモ（スタート・完了あり）

| 用語 | 意味 |
|------|------|
| **スタート時点理論** | そのセッションで「スタート」を押した時点の現在庫（JAN別で記録）。セッション中は一切変更しない。 |
| **セッションの棚卸実数** | そのセッションでスキャンして登録した数量の合計（JAN別）。完了で確定。 |
| **保存データ** | セッションごとに「開始日時」「完了日時」「スタート時点理論（JAN別）」を保持。スキャン行にセッションIDを紐づけるか、完了日時でセッションを識別。 |
| **画面** | 棚卸画面に「スタート」「完了」ボタン。期間中は「棚卸実施中」などの表示。差異一覧は完了セッションの スタート時点理論・棚卸実数・差異 を表示。 |
