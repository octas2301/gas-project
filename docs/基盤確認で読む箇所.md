# 基盤の確認で読む箇所（3-2 用）

いま一度、基盤を確認したい場合に**順に読むファイルと見出し**です。問題なければ「基盤OK」「承認しました」と返信してください。承認済みなら §7 の承認者欄に「承認済」と記載します。

---

## 1. 読む対象と該当箇所

### 1) 要件定義（約75行）

| 項目 | 内容 |
|------|------|
| **ファイル** | `docs/REQUIREMENTS.md` |
| **読む範囲** | **全文**（1〜6領域のタスク表と優先度メモ） |
| **主な見出し** | §1 リサーチ・見積もり、§2 出品、§3 販売拡大、§4 商品管理、§5 顧客対応、§6 財務・経理、および各領域内のタスク表 |

---

### 2) Runbook（約50行）

| 項目 | 内容 |
|------|------|

| **主な見出し** | 日次タスク、週次タスク、隔週タスク、月次タスク、運用メモ |

### 3) フロー・優先順位（本ドキュメント）

| 項目 | 内容 |
|------|------|
| **ファイル** | `docs/FLOW_AND_PRIORITY.md` |
| **読む範囲** | 次の **§（セクション）** のみ。順に読む。 |
| **§1** | **フロー間の接続と必須要件**（表：接続ごとの必須要件・細かくするタイミング） |
| **§2** | **クリティカルパス**（商品が売れるまでの経路・今回決めたこと） |
| **§3** | **自動化構築の優先順位**（表：優先順位1〜8） |
| **§4** | **優先順位2の具体化**（入力元の選択肢・最小必須項目の表・既存マスタ連携の対応表） |
| **§6** | **基盤の確認で読む箇所**（この一覧の元になった表） |
| **§7** | **次のアクション**（誰が・いつまでに・何をするか。承認者欄あり） |

※ §5（すり合わせの進め方）・§8（優先順位3以降の進め方・開発効率優先の実装順）は任意。流れを把握したいときだけ読めば可。

---

## 2. 確認後の返答例

- 「基盤OK」「承認しました」→ 承認済みとして記録します。  
  **2026年2月**: 再確認のうえ「承認しました」をいただき、FLOW_AND_PRIORITY.md §7 に反映済み。
- 変更したい点があれば、該当ファイル名と「どの部分をどうしたいか」を短文で書いてください。
