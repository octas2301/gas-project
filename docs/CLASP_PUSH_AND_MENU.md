# clasp push とメニュー反映

## 重要：1回の clasp push で更新されるのは「1つのプロジェクト」だけ

- **clasp push** は、**今いるフォルダの .clasp.json に書かれた scriptId の、その1つの GAS プロジェクト**にだけ反映します。
- **gas-project** のルートで `clasp push` すると、**AI出品ツール用のプロジェクト（メイン）** だけが更新されます。
- **広告運用** は **別の scriptId の別プロジェクト** なので、**gas-project で push しても広告運用側は更新されません**。
- 逆に、**広告運用を更新したいとき**は、[広告運用GAS_push手順.md](広告運用GAS_push手順.md) のとおり **Desktop の 広告運用GAS_clone にコピーしてから、そのフォルダで clasp push** する必要があります。

**「同時に両方のプロジェクトを反映」はできません。** メインと広告運用は **別々に**、それぞれの push 用フォルダで push します。

### もし「Pushed ○ files」に inventory-app\… や docs\… や 参考資料\… が含まれていたら

- それらは **メインプロジェクト（AI出品ツール）に誤って取り込まれた**状態です。別プロジェクトのコードが混ざるとメニューや動作がおかしくなります。
- **今すぐやること**は下記「誤 push したときの対処（メインプロジェクトを元に戻す）」を参照してください。
- **再発防止**：ルートの **`.claspignore`** に、サブプロジェクトやドキュメントのフォルダ（`inventory-app/**`、`docs/**` 等）を必ず追加すること。下記「新しいプロジェクトを追加したときにやること」を参照。

## 誤 push したときの対処（メインプロジェクトを元に戻す）

**やること**: [script.google.com](https://script.google.com) で **メインプロジェクト（AI出品ツール）** を開き、**コード.js と Yahoo.js 以外のファイル・フォルダを削除**してから、正しいコードだけを再度 push する。

### 手順

1. **[script.google.com](https://script.google.com)** にアクセスし、**AI出品ツール用のプロジェクト**（メイン）を開く。

2. **左のファイル一覧**を確認する。  
   **メインプロジェクトの正しい構成は「コード.js」と「Yahoo.js」の 2 ファイルだけ**です（.gs と表示される場合もあります）。

3. **次のものをすべて削除する**（右クリック → 削除）。
   - **inventory-app** フォルダ（中身ごと）
   - **docs** フォルダ
   - **参考資料** や **参考資料：〇〇** といったフォルダ
   - **広告運用GAS** フォルダ（あれば）
   - 上記以外で **「コード.js」「Yahoo.js」以外の .gs / .js / .html** で、メインのAI出品ツール用でないもの

4. **「コード.js」** を開き、中身が **AI出品ツール** のコード（`createMenu('AI出品ツール')` や `計算式を書き出し` などがある）か確認する。  
   別プロジェクトのコードになっていたら、ローカルの **gas-project\コード.js** の内容をコピーして貼り付け、保存する。

5. **.claspignore を確認**する（ローカル）。  
   `inventory-app/**` と `docs/**` と `参考資料/**` 等が含まれているか確認。含まれていなければ追加して保存（今回の修正で追加済み）。

6. **再度 push** する。
   ```powershell
   cd C:\Users\takuy\Desktop\gas-project
   clasp push
   ```
   **「Pushed 2 files」**（コード.js と Yahoo.js のみ）と出ればOK。  
   その後、スプレッドシートを **F5 で再読み込み**し、メニューが「AI出品ツール」になっているか確認する。

## メニューが「広告運用メニュー」になってしまったとき（元に戻す）

**原因**: 誤って 広告運用GAS のファイルがメインプロジェクトに push され、広告運用の `onOpen()` がメニューを「広告運用メニュー」で上書きしている状態です。

**やること**: メインプロジェクトから **広告運用用のファイルを削除**し、**正しいメインのコードだけ**にしたあと、必要なら **再度 push** してスプレッドシートを再読み込みします。

### 手順

1. **自動出品のスプレッドシート**を開いた状態で、メニュー **拡張機能** → **Apps Script** を開く。  
   （または [script.google.com](https://script.google.com) にアクセスし、左の「マイプロジェクト」から **このスプレッドシートに紐づいているプロジェクト** を開く。）

2. **左のファイル一覧**を確認する。  
   メインプロジェクトの正しい構成は **「コード.js」と「Yahoo.js」の 2 ファイルだけ**です（名前は コード.js / Yahoo.js のまま、または コード.gs / Yahoo.gs のように表示される場合があります）。

3. **次のファイルがあれば削除する**（右クリック → 削除、またはファイルを開いてメニューから削除）。
   - **広告運用GAS** フォルダ（または **広告運用GAS\コード.js** など、広告運用用のファイル）
   - **参考資料** や **.html** など、GAS コードではないファイル
   - 上記以外で **「コード.js」「Yahoo.js」以外の .gs / .js ファイル**で、明らかに広告運用用のもの

4. **「コード.js」（または コード.gs）** を開き、中身が **AI出品ツール** のコード（`createMenu('AI出品ツール')` や `計算式を書き出し` などがある）か確認する。  
   もしここが広告運用のコード（`広告運用メニュー` など）になっていたら、**ローカルの gas-project の コード.js を開いて、その内容をコピーし、Apps Script の コード.js に貼り付けて上書き保存**する。

5. **保存**（Ctrl+S）したあと、**自動出品のスプレッドシート**に戻り、**F5 で再読み込み**する。  
   メニューが **「AI出品ツール」** に戻っているか確認する。

6. まだ「広告運用メニュー」のままなら、**gas-project で再度 push** してメインプロジェクトを上書きする。  
   （`.claspignore` 更新済みなので、今度は コード.js と Yahoo.js だけが push されます。）
   ```powershell
   cd C:\Users\takuy\Desktop\gas-project
   clasp push
   ```
   その後、スプレッドシートを再読み込み（F5）する。

## 「A file with this name already exists in the current project: appsscript」が出るとき

**原因の候補**

- サーバー側に **appsscript** という名前のファイルが既にあり、ローカルの **appsscript.json** を push すると名前が衝突する。
- **広告運用GAS** と同様に、**gas-project の内側**で push すると、clasp が親フォルダの設定を参照して競合することがある（[広告運用GAS_push手順.md](広告運用GAS_push手順.md) 参照）。

**対処（順に試す）**

1. **appsscript.json を push しない**  
   プロジェクトの **`.claspignore`** に **`appsscript.json`** を追加済みです。**コード（.js）だけ** push するようにして、マニフェストの衝突を避けます。  
   → `clasp push` または `clasp push --force` を再度実行する。

2. **それでも同じエラーになる場合：Desktop の「専用 push 用フォルダ」で push する**  
   広告運用と同様に、**編集は gas-project、反映は別フォルダで push** する手順を下に記載しています。

## ASIN貼り付けシートでKeepa取得後にシートが更新されないとき（調査ログの確認）

Keepa取得を実行してもシートに商品名・価格・セット数が書き込まれない場合、次の手順で**実行ログ**を確認できる。

1. **拡張機能** → **Apps Script** でプロジェクトを開く。
2. 左メニュー **「実行数」**（または **トリガー** の上あたりの **「実行」**）をクリック。
3. 直近の **「Keepa取得」実行** をクリックして詳細を開く。
4. **「ログ」** タブを開く。次のような `[Keepa]` ログが出る。
   - **レイアウト**: ヘッダー行・ASIN列・商品名/競合価格/セット数の列番号
   - **ブロックN**: 検出したASIN件数・先頭3件のASIN
   - **API応答**: Keepa API が返した件数（null ならキー・ネットワーク・トークン要確認）
   - **書き込み**: 各ブロックで何件書き込んだか

**想定される原因**

- **「ASIN」ヘッダーが見つかりません** → 1行目または2行目付近に「ASIN」というセルが無い。列名を「ASIN」に合わせるか、標準レイアウト（2行目が ASIN, 商品名, 評価, 競合価格(Amazon), セット数, 商品URL）にする。
- **API応答: null** → Script Properties の `KEEPA_API_KEY` 未設定・誤り、またはネットワーク・トークン残数。
- **書き込み=0件** → APIは返しているがASINが一致していない（ログの「含まれずスキップ」を確認）。

**シートのレイアウト**

- **1行目がヘッダー**（商品ID, ASIN, 商品名, 競合価格(Amazon), セット数, メモ など）でも対応済み。先頭6行のどこかに「ASIN」列があれば検出し、列名「商品名」「競合価格(Amazon)」「セット数」があればそこに書き込む。

## Keepa取得でソート失敗したとき（ソート範囲パラメータの確認手順）

「サービス エラー: スプレッドシート」でソートが失敗する場合、**範囲パラメータがログに出力されている**ので、次の手順で値を確認できる。

### 手順

1. **Keepa取得**を実行する（メニュー「商品リサーチ」→「② 出品用」→「ASIN貼り付け（Keepa用）を準備」でシートを開き、ASINを貼り付けたうえで「Keepa取得」を実行）。
2. **拡張機能** → **Apps Script** でプロジェクトを開く。
3. 左メニュー **「実行数」**（または **「実行」**）をクリックし、直近の **Keepa取得** の実行を選んで詳細を開く。
4. **「ログ」** タブを開き、`[Keepa] ソート範囲` で検索する。
5. 次のような1行が出力されている。
   - **firstRow1Based** … データ先頭行（1-based）。通常は 3。
   - **numRows** … そのブロックで書き込んだ行数（例: 20）。
   - **startCol1Based** … ブロック先頭列（1-based）。ブロック1なら 1、ブロック2なら 11。
   - **endCol1Based** … ブロック末尾列（1-based）。ブロック1なら 10。
   - **numCols** … 列数（endCol1Based − startCol1Based + 1）。通常は 10。

**確認のポイント**

- 結合セルがソート範囲（上記の行・列の矩形）にかかっていないか、シート上で目視する。
- `numRows` が 0 や 1 でないか（2行以上ないとソートしない）。
- 他ブロックの列と重なっていないか（例: ブロック1は 1〜10、ブロック2は 11〜20）。

ログの値が想定と違う場合は、ASIN貼り付けシートのレイアウト（ヘッダー行・ASIN列の位置）を確認する。

## 新しいプロジェクトを追加したときにやること（再発防止）

**gas-project のルートで `clasp push` すると、ルートの .clasp.json の scriptId ＝ メインプロジェクトだけに反映されます。**  
ただし **.claspignore に書いていないフォルダはすべて push 対象**になるため、サブプロジェクトや docs を入れたまま push すると、メインに誤って取り込まれます。

**ルール：新しいサブプロジェクト（別のGAS用フォルダ）や「push したくないフォルダ」を gas-project 内に作ったら、必ずルートの `.claspignore` に 1 行追加する。**

| 追加したフォルダ | .claspignore に追加する行 |
|------------------|---------------------------|
| 新しいGAS用サブプロジェクト（例: my-new-app） | `my-new-app/**` |
| ドキュメント用（docs） | `docs/**` |
| 参考資料など | `参考資料/**` や `参考資料：*/**` |

**今回の修正でルートの .claspignore に追加したもの**  
`inventory-app/**`、`docs/**`、`参考資料：*/**` を追加済み。今後もサブプロジェクトを増やしたらここに追記すること。

## メニューに新しい項目が現れないとき

GAS のカスタムメニュー（例: **AI出品ツール**）は、**スプレッドシートを開いたときに `onOpen()` が実行されて**つくられます。

- **clasp push が成功しても、その時点で開いているタブではメニューは更新されません。**
- **スプレッドシートを再読み込みする**と、`onOpen()` が再度実行され、最新のメニューが表示されます。

**手順**

1. **clasp push**（必要なら `clasp push --force`）でコードを反映する。
2. **反映が成功したら**、該当のスプレッドシートのタブで **F5 で再読み込み**するか、**タブを閉じて開き直す**。
3. メニュー「**AI出品ツール**」を開き、「**計算式を書き出し（表示中シート）**」などが表示されるか確認する。

## 対処2：Desktop の専用フォルダで push する（広告運用と同様）

**「A file with this name already exists: appsscript」が .claspignore 後も出る場合**は、[広告運用GAS_push手順.md](広告運用GAS_push手順.md) と同様に、**一度 Desktop のフォルダにコピーしてからそこで push** する。

**流れ**

1. 編集は **gas-project** で行う（いつも通り）。
2. 反映するときだけ **必要なファイルを Desktop の push 用フォルダにコピー**する。
3. **そのフォルダで `clasp push`** する。

### 手順（毎回）

PowerShell で次を **順に** 実行する。

```powershell
# 1. gas-project の コード.js と Yahoo.js を Desktop の push 用フォルダにコピー
$dest = "C:\Users\takuy\Desktop\gas-project_main_push"
New-Item -ItemType Directory -Force -Path $dest | Out-Null
Copy-Item -Path "C:\Users\takuy\Desktop\gas-project\コード.js" -Destination $dest -Force
Copy-Item -Path "C:\Users\takuy\Desktop\gas-project\Yahoo.js" -Destination $dest -Force
Copy-Item -Path "C:\Users\takuy\Desktop\gas-project\.clasp.json" -Destination $dest -Force

# 2. そのフォルダに移動して push
cd $dest
clasp push
```

- **「Updated 2 files.» など** と出れば、AI出品ツール用の Apps Script に反映されている。
- 反映後は **スプレッドシートを再読み込み（F5）** すると、メニューが更新される。

### push 用フォルダを初めて作る場合

Desktop に **gas-project_main_push** がない、または空の状態から用意する場合。

```powershell
cd C:\Users\takuy\Desktop
mkdir gas-project_main_push -ErrorAction SilentlyContinue
cd gas-project_main_push
# 既に clone 済みで .clasp.json がある場合は上記「手順（毎回）」のコピー＋push でよい。
# 初回のみ clone する場合（scriptId は gas-project の .clasp.json と同じもの）
clasp clone "1DEMdFJKHVY7IRdncPOkxnasEipjmYDx_MJMn1vcDm5iUw2g5cQZhLKis"
```

clone すると **リモートのファイルが降りてくる** ため、以降は「手順（毎回）」で **gas-project の コード.js と Yahoo.js を上書きコピーしてから push** する運用にする。
