# CPO値決め（Gemini）— マッピング・実行単位・書き込み先

## 1. マッピングの場所

プレースホルダ ⇔ マスタ列の対応は **00_設定マスタ** の **A92～C100** に記載する。  
GASはこの範囲を読み、プロンプトの `{{ }}` を埋める際の参照列として使用する。

- A列: プレースホルダ名（例: 商品名、手数料率）
- B列: マスタ列名（例: 商品名、amazon手数料率）
- C列: 備考（親行/子行、単位など）

## 2. 実行単位・対象

- **対象**: マスタで **出品CK が TRUE の行** に属する **JAN（商品）** をすべて対象とする。
- **実行方法**: **1商品（1JAN）ごと** に実行する。レ点の付いた商品が複数ある場合は、順番に1つずつ処理し、プロンプト組み立て → Gemini呼び出し → 結果の書き込み を繰り返す。
- 同一JANの親行＋子行を1単位として、そのJANのデータだけを集約してプロンプトを組み立てる。

## 3. 書き込み先

| 内容 | 書き込み先 | 備考 |
|------|------------|------|
| CPOの返答本文（戦略マトリクス・解説。JSONブロックは除く） | **親SKU行** の列 **「amazon価格戦略」** | 1セルに長文。人間が「なぜその価格か」を後から確認できるようにする。実装では JSON ブロックを除いた本文のみ書き込む（§6 実装メモ）。 |
| 販売価格（数値） | 各 **子行** の列 **「販売価格amazon」** | Gemini返答のJSONをパースし、セット数に対応する行に書き込む。 |
| amazon手数料計 | **書き込まない** | マスタで計算式により算出しているため、GASでは更新しない。 |

## 4. プロンプト本文

- 参照: `docs/CPO_PROMPT.md`
- GASでは同一内容を定数または文字列として保持し、`{{ }}` をマスタから取得した値で置換する。

## 5. 楽天・Yahoo! 拡張

- 出力JSONの `rakuten` / `yahoo` は将来用。今回対象は Amazon のみで、`amazon` 配列のみパースして書き込む。
- 同じプロンプトで楽天・Yahoo! を渡す場合は、入力の `{{ モール名 }}` とセット数別テーブルの競合列を切り替え、JSONの `rakuten` / `yahoo` にも配列を出力させる。

---

## 6. 実装メモ（2026-03）

- **親行の amazon価格戦略**: Gemini の返答のうち **JSON ブロック（## 5. 【JSONデータ】と ```json ... ```）を除いた本文のみ**を書き込む。人間がセルを読むとき JSON が不要なため。
- **パース**: 価格反映用のデータは、返答テキスト内の **先頭の ```json ブロック** から取得する。JSON ブロックが無い場合は **Markdown 表**（戦略的価格マトリクス）から setCount と price を抽出するフォールバックでパースする。
- **実行前チェック**: 子SKUの「サイズ＆自己発/FBA」「梱包箱指定」が空白の JAN が1件でもある場合、実行前にポップアップで「送料は入力されていますか？ はい／いいえ」を表示する。いいえを選んだ場合は実行しない（送料未入力でのやり直しを防ぐ）。
- **原因追跡用ログ**: JAN ごとに、マスタに存在するセット数一覧・パース結果のセット数一覧を Logger で出力する。さらに、パース結果に含まれず価格が反映されなかった子行（setNum）について「JAN=xxx setNum=N に価格を反映できませんでした（パース結果に setCount=N が含まれていません）」をログ出力する。
