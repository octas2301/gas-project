# メニューとフロー サマリー

主要メニューを連番で整理し、メニュー名の案と人間の作業・その他個別メニューをまとめたもの。

---

## 1. 主要メニュー一覧（連番・提案メニュー名）

| 連番 | 提案メニュー名 | 対応する現状の処理／備考 |
|------|----------------|---------------------------|
| ①-1 | Keepa取得（20件） | ASIN貼り付けシートでKeepa取得（20件まで）※準備は統合済 |
| ①-2 | Keepa取得（50件） | ASIN貼り付けシートでKeepa取得（50件まで） |
| ①-3 | Keepa取得（全件） | ASIN貼り付けシートでKeepa取得（全件） |
| ② | amazonセット構成提案 | セット構成提案 |
| ③ | amazon販売価格を提案（CPO） | ③販売価格の調整　CPOで価格提案（Gemini） |
| ④ | 楽天Yahoo!リサーチ一括（競合価格⇒セット数抜け追加） | モール横断 → 抜け挿入 → 反映（**価格提案は含めない**） |
| ⑤ | 楽天Yahoo!販売価格を提案（CPO） | 選択行に価格・セット数提案を反映（楽天・Yahoo!・Amazonの販売価格を一括提案）※マスタで対象行を選択してから実行 |
| ⑥ | 出品CKをクリア | 出品CKをクリア（選択行） |
| ⑨ | その他個別メニュー | 運用ではほぼ使わないが、不具合時の検証用に残す（下記で分類） |

**④と⑤を分ける理由**  
④のあと、人間がマスタで競合価格・セット数を確認・修正してから、⑤で販売価格を提案する流れにするため。⑤は「マスタを開き、対象行を選択した状態で」実行する。

## 2. ⑨ その他個別メニュー（検証用）

### Amazon

| メニュー名 | 用途 |
|------------|------|
| ASIN貼り付け（Keepa用）シートを準備 | シートが無いときの作成・切り替えのみ（①で統合済のため単体は補助） |
| ASIN貼り付けシートをリセット | Keepa用シートのリセット |
| 競合・穴のセット数を提案（ASIN貼り付け） | ASIN貼り付けシート右端に競合・穴のセット数を表示（②でマスタには書くので省略可） |
| 競合価格のみ修正を反映 | ASIN貼り付けで直した競合価格・ASIN・URLをマスタに反映 |
| 販売価格を提案（セット構成提案後に実行） | ASIN貼り付けの天井で販売価格を算出（③CPOのみで運用する場合は不要） |
| 選択行のASINでKeepaから競合価格を取得 | 単品でKeepa競合価格を取得 |
| リサーチCSVから競合価格Amazonをマスタに反映 | CSV取り込み用 |

### 楽天

| メニュー名 | 用途 |
|------------|------|
| 選択行のJANで楽天から競合価格を取得 | 単品で楽天競合価格を取得 |
| 選択行のJANで楽天からセット別競合価格を取得 | セット別楽天競合のみ取得 |

### Yahoo!

| メニュー名 | 用途 |
|------------|------|
| 選択行のJANでYahoo!から競合価格を取得 | 単品でYahoo!競合価格を取得 |
| 選択行のJANでYahoo!からセット別競合価格を取得 | セット別Yahoo!競合のみ取得 |

### 共通

| メニュー名 | 用途 |
|------------|------|
| 選択行に競合価格を入力 | 手動で入力した競合価格の一括反映 |
| モール横断 セット数統合判定 | ④の1ステップ目のみ単体実行 |
| モール横断結果を楽天・Yahoo! マスタに反映 | ④の反映のみ単体実行 |
| 抜けセット数行の挿入 | ④の抜け挿入のみ単体実行 |
| 選択行に価格・セット数提案を反映 | ⑤と同じ処理の単体実行 |
| Amazonカテゴリーを自動入力（Gemini） | カテゴリ列の自動入力（②の最後でも実行される） |

※不要と判断したものは削除してよい。残す場合は「⑨その他個別メニュー」の下に Amazon／楽天／Yahoo!／共通 でまとめておくと検証しやすい。

## 3. 人間がやること（入力・チェック・設定）

### A. Keepa起点（Amazon）フロー

| 順 | どこで | 何をする |
|----|--------|----------|
| 事前 | AI情報取得data | メーカー名・仕入判断・JAN・商品名・卸値・ASINコード・参考画像情報などを入力 |
| 事前 | ASIN貼り付け（Keepa用） | 品番（ブロック）ごとにASINを入力 |
| ①の後 | ASIN貼り付け（Keepa用） | セット数・価格・ASINが正しいか確認（◎付与など） |
| ②の後 | ▼商品マスタ(人間作業用) | セット数・競合価格を確認 |
| ②の後 | ▼商品マスタ(人間作業用) | 送料を設定（③の前に必須） |
| ③の後 | ▼商品マスタ(人間作業用) | 販売価格amazon・価格戦略を確認し、必要なら修正 |
| 出品後 | ▼商品マスタ(人間作業用) | 対象行を選択して ⑥出品CKをクリア を実行 |

### B. 楽天・Yahoo!（モール横断）フロー

| 順 | どこで | 何をする |
|----|--------|----------|
| ④の前 | ▼商品マスタ(人間作業用) | 対象行（レ点など）を決めておく（④は行選択不要で実行可） |
| ④の後 | ▼商品マスタ(人間作業用) | 競合価格・セット数が正しいか確認し、必要なら修正 |
| ⑤の前 | ▼商品マスタ(人間作業用) | **対象行を範囲選択した状態で** ⑤楽天Yahoo!販売価格を提案（CPO） を実行 |
| ⑤の後 | ▼商品マスタ(人間作業用) | 楽天価格設定・Yahoo!価格設定・販売価格amazon を確認し、必要なら修正 |
| 出品後 | ▼商品マスタ(人間作業用) | 対象行を選択して ⑥出品CKをクリア を実行 |

---

## 4. 実行順の要約

**A. Keepa起点（Amazon）**  
事前入力（AI情報取得data・ASIN貼り付け） → ①-1／①-2／①-3 → [チェック①] → ②amazonセット構成提案 → [チェック②・送料設定] → ③amazon販売価格を提案（CPO） → [チェック③] → 出品 → ⑥出品CKをクリア

**B. 楽天・Yahoo!（モール横断）**  
④楽天Yahoo!リサーチ一括（競合価格⇒セット数抜け追加） → [競合・セット数のチェック・修正] → [対象行を選択] → ⑤楽天Yahoo!販売価格を提案（CPO） → [価格のチェック・修正] → 出品 → ⑥出品CKをクリア

## 5. 実装メモ

- **④** … 「リサーチ一括」から「選択行に価格・セット数提案を反映」の呼び出しを外し、モール横断→抜け挿入→反映の3ステップに変更。
- **⑤** … メニュー「楽天Yahoo!販売価格を提案（CPO）」で既存の「選択行に価格・セット数提案を反映」を実行。
- **①** … Keepa取得実行時にシートが無ければ自動作成してから取得処理へ続行（準備メニューは⑨に移動）。
- **⑨** … サブメニュー「⑨その他個別メニュー」の下に Amazon／楽天／Yahoo!／共通 を配置。
