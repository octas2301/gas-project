## 隔週金曜 3:00 トリガーの設定手順

1. **広告運用スプレッドシート** を開く → **拡張機能** → **Apps Script**。
2. 左の **トリガー（時計アイコン）** をクリック → **トリガーを追加**。
3. 次のように設定する：
   - **実行する関数**: `runOptimizationAndBulkOnSchedule`
   - **実行するデプロイ**: ヘッド
   - **イベントのソース**: 時間駆動型
   - **時間ベースのトリガー**: **日タイマー** → **午前3時～4時**
4. **保存** する。

※ 関数内で **「今日が金曜かつ第2 or 第4金曜」** のときだけ分析＆バルク生成を実行します。それ以外の日は何もせず終了します。  
※ 実行時は **指定フォルダに Amazon・楽天のCSVが事前に入っている必要**があります（手動配置 or 今後の ① ② で自動取得）。

## Amazon API メモ（セラーセントラル・広告からの情報取得）

### 広告レポート（スポンサー広告）

- **Amazon Advertising API** で、スポンサー広告のレポート（検索ワードレポート等）を **API経由で取得** できます。
- 認証は **LWA（Login with Amazon）＋ リフレッシュトークン** が一般的です。広告管理画面で「API 連携」や「開発者向け」からアプリ／認証情報を発行します。
- GAS からは **UrlFetchApp** で API を呼び、取得したJSON/CSV を Drive の指定フォルダに保存する形で自動化できます。
- 参考：テンプレートやアドオンで「Amazon 広告レポート → スプレッドシート」を自動取得する事例があります（TēPs など）。自前実装する場合は [Amazon Advertising API ドキュメント](https://advertising.amazon.com/API/docs) でレポート取得 API を確認します。

### セラーセントラル（出品・在庫など）

- **SP-API（Selling Partner API）** がセラーセントラル側の公式APIです。在庫・注文・レポートなどが取得できます。
- **広告運用で使う「検索ワードレポート」** は **Advertising API** の対象で、**SP-API とは別**です。広告レポートの自動取得は **Advertising API** を使います。
- SP-API は OAuth や IAM ロールなど認証がやや複雑なため、まずは **広告レポートの自動取得（Advertising API）** から進めるのが扱いやすいです。

### まとめ

| 取得したい情報 | 使うAPI | 備考 |
|----------------|---------|------|
| 広告レポート（検索ワード・キャンペーン等） | **Amazon Advertising API** | ① ② の「レポート自動取得」はここで実装可能。 |
| 在庫・注文・出品情報 | **SP-API（Selling Partner API）** | 広告運用の ① とは別。必要なら別途設計。 |

① ② を進める際は、**Amazon Advertising API** のレポート取得仕様・申請手順・レートリミットを確認し、GAS から「指定フォルダにCSV保存」する流れを要件化します。
