# 在庫管理アプリ — 次のテスト手順

メニュー移動ができるようになったあと、各機能を順に確認するためのチェックリストです。

---

## テストの進め方

- **1項目ずつ** 実行し、期待どおりかメモする。
- エラーや想定外の動作があれば、**画面の様子** と **F12 Console のエラー** を控える。
- スプレッドシートの **該当シート** にデータが入っているかもあわせて確認するとよい。

---

## 1. メニュー移動（実施済み）

- [x] 出荷 → 移動 → 入荷 → 棚卸 → 差異 → 商品 → 在庫 → 設定 の順にタップし、それぞれの画面が表示される。
- [x] 何度切り替えても白画面にならない。

---

## 2. 出荷（1点登録）

**手順**

1. メニューで **「出荷」** を開く。
2. **JANコード** に、商品マスタに登録済みの JAN（またはこれから登録する JAN）を入力する。
3. **出荷元** で場所を選択する。
4. **「登録（1点）」** をクリックする。

**確認**

- [ ] 「登録しました」などのトーストが表示される（またはエラー文が表示される）。
- [ ] スプレッドシートの **「出荷リスト」** シートに 1 行追加されている。
- [ ] 未登録の JAN の場合は、仕様に応じてエラーまたは「未登録」表示になる。

---

## 3. 商品登録（出荷・入荷の前提）

**手順**

1. メニューで **「商品」**（🛠️ 商品情報）を開く。
2. **単品JAN** にテスト用 JAN（例: 13桁の数字）を入力する。
3. **商品名** を入力する（任意で箱JAN・原価・箱入数も）。
4. **「登録」** をクリックする。

**確認**

- [ ] 「登録しました」が表示される。
- [ ] スプレッドシートの **「商品マスタ」** に 1 行追加されている。
- [ ] 出荷画面で同じ JAN を入力すると、商品名が表示される（JAN 入力後の blur で取得）。

---

## 4. 移動（在庫移動）

**手順**

1. メニューで **「移動」** を開く。
2. **JANコード** に商品マスタにある JAN を入力する。
3. **移動元**・**移動先** を選択し、**移動数量** を入力する。
4. **「登録」** をクリックする。

**確認**

- [ ] 「登録しました」が表示される。
- [ ] スプレッドシートの **「移動ログ」**（または仕様で定めたシート）に 1 行追加されている。

---

## 5. 入荷

**手順**

1. メニューで **「入荷」** を開く。
2. **JANコード** を入力する。
3. **仕入単価**・**箱入数**・**箱数**・**端数** を入力する（数量は自動計算される）。
4. **ロケーション** と **入荷理由** を選択する。
5. **「登録」** をクリックする。

**確認**

- [ ] 「登録しました」が表示される。
- [ ] スプレッドシートの **「入荷ログ」** に 1 行追加されている。

---

## 6. 棚卸

**手順**

1. メニューで **「棚卸」** を開く。
2. **JANコード** を入力する。
3. **箱入数**・**箱数**・**端数** を入力する（数量 ＞ 0 にする）。
4. **ロケーション** を選択する。
5. **「保存」** をクリックする。

**確認**

- [ ] 「保存しました」が表示される。
- [ ] スプレッドシートの **「棚卸スキャンリスト」** に 1 行追加されている。
- [ ] **棚卸の箱入数**：JAN を入力して他フィールドにフォーカスを移すと、商品マスタの箱入数が自動で入る。そのままでも、手で修正してもよい。

---

## 7. 差異

**手順**

1. メニューで **「差異」** を開く。

**確認**

- [ ] 理論在庫と棚卸実数の差（0 でないもの）の一覧が表示される、または「差異はありません」と表示される。
- [ ] エラーにならずに画面が表示される。

---

## 8. 現在庫

**手順**

1. メニューで **「在庫」**（📊 現在庫）を開く。

**確認**

- [ ] 前回棚卸＋入荷－出荷（全社合計）の一覧が表示される、または「データがありません」と表示される。
- [ ] エラーにならずに画面が表示される。

---

## 9. 設定（担当者）

**手順**

1. メニューで **「設定」** を開く。
2. **現在の担当者** でドロップダウンから 1 件選ぶ。
3. **「保存」** をクリックする。

**確認**

- [ ] 「保存しました」が表示される。
- [ ] スプレッドシートの **「設定」** シート（または担当者を保存するシート）に値が反映されている。

---

## 10. Console のエラーについて

- **Unrecognized feature**（ambient-light-sensor など）や **iframe sandbox** の警告は、GAS の iframe 環境によるもので、多くの場合は無視してよい。
- **TypeError: Cannot set properties of null** や **message channel closed** が、操作のたびに毎回出る場合は、その操作手順と一緒にメモしておくと原因切り分けに役立つ。
- **error: AmazonProductAccessUserOverlayAsyncVM** はブラウザ拡張機能由来の可能性が高く、在庫アプリ本体の不具合とは限らない。

## まとめ

| 順番 | 項目       | 主な確認内容                         |
|------|------------|--------------------------------------|
| 1    | メニュー移動 | 全画面表示・白画面なし               |
| 2    | 出荷       | 登録 → 出荷リストに 1 行             |
| 3    | 商品登録   | 登録 → 商品マスタに 1 行・出荷で商品名表示 |
| 4    | 移動       | 登録 → 移動ログに 1 行               |
| 5    | 入荷       | 登録 → 入荷ログに 1 行               |
| 6    | 棚卸       | 保存 → 棚卸ログに 1 行               |
| 7    | 差異       | 一覧表示 or 「差異はありません」     |
| 8    | 現在庫     | 一覧表示 or 「データがありません」   |
| 9    | 設定       | 担当者保存 → 設定シートに反映        |

まずは **2. 出荷** と **3. 商品登録** から試すと、データの流れを確認しやすいです。商品マスタに 1 件入れてから、その JAN で出荷を 1 件登録し、スプレッドシートの各シートを確認してください。

## 次のテスト（出荷・入荷・棚卸まで完了した場合）

ここまで動いているときは、次を順に確認してください。

| 順番 | テスト項目 | やること・確認 |
|------|------------|----------------|
| **7** | **差異** | メニューで「差異」を開く → 理論在庫と棚卸実数の差（0 でないもの）の一覧が出る、または「差異はありません」と表示される。エラーにならないこと。 |
| **8** | **現在庫** | メニューで「在庫」を開く → 一覧 or 「データがありません」が表示される。エラーにならないこと。 |
| **9** | **設定** | 担当者を選んで「保存」→ 「保存しました」が出て、必要なら設定シートに反映されていること。（※既に実施済みならスキップ可） |

**棚卸の箱入数について**  
棚卸画面で JAN を入力し、次の欄にカーソルを移す（blur）と、商品マスタに登録されている **箱入数** が「箱入数」欄に自動で入ります。その値はそのまま使っても、手入力で変更してもかまいません。

**差異・現在庫の数値について**  
- **現在庫** = 前回棚卸日の棚卸実数 ＋ その後入荷 － その後出荷（全社合計）。  
- **差異** = 今月の棚卸実数 － 理論在庫（理論在庫＝現在庫と同じ）。  
  - 差異が 0 でないものだけ一覧に表示されます。  
- 「差異数が違う」と感じた場合、**棚卸スキャンリスト**の「年月」が日付型で入っていると、今月分が集計されず棚卸＝0 になる不具合がありました。コードで年月の比較を修正済みなので、`clasp push` とデプロイ更新後に再度ご確認ください。必要なら後から仕様（集計ロジック）の調整も可能です。
