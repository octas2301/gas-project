# スマホでバーコードスキャン＋タップで進める UI のイメージ

## 1. 運用イメージ（できること）

**「スマホでバーコードをスキャンし、タップしながら入力を進めていく」運用** は、GAS で配信する Web アプリ（HTML）で実現できます。

- 画面を開くと **JAN 用の入力欄にフォーカス** が当たる（または「スキャン」ボタンでフォーカスを当てる）。
- **バーコードスキャン** すると、その入力欄に JAN が入る（端末のバーコードキーボード／外付けスキャナを使用）。
- スキャン後、**次の項目（数量・場所など）へタップで移動** し、必要なら値を変更してから **保存**。
- 保存後は **同じ画面のまま、再び JAN 欄にフォーカス** を戻し、次のスキャンに備える（連続スキャン）。

この「スキャン → タップで次へ → 保存 → 次のスキャン」の流れを、1 画面で繰り返せる UI にします。

---

## 2. 実現のポイント（技術的に可能な理由）

| ポイント | 説明 |
|----------|------|
| フォーカス制御 | ページ表示時や保存後に `input.focus()` で JAN 欄にフォーカスを当てられる。スキャン対応キーボードなら、その入力欄に JAN が直接入る。 |
| タップで次へ | 入力欄を大きめにし、タップしやすいブロックにする。必要なら「次へ」ボタンで次の input にフォーカスを移す（`Advance forms automatically` に相当）。 |
| 1 画面で完結 | 出荷・入荷・棚卸などは、それぞれ 1 画面に「JAN → 商品名（表示）→ 数量・場所等 → 保存」を並べ、送信後は同じ画面を維持して JAN だけクリアし、フォーカスを JAN に戻す。 |
| スマホで使いやすく | ボタン・入力欄を十分なサイズにし、固定フッターでメニューを常に表示。レスポンシブ CSS でスマホ縦画面を前提にレイアウトする。 |

※ カメラでバーコードを読み取る方式は、別途 JavaScript ライブラリ等の検討が必要です。まずは **スキャナ／バーコード入力キーボード** で JAN を入力する方式であれば、上記の UI で問題なく実現できます。

## 3. 画面フロー（例：棚卸）

```
┌─────────────────────────────────────┐
│  棚卸                              │
├─────────────────────────────────────┤
│                                     │
│  [JANコード]  ★ここにフォーカス     │
│  ┌─────────────────────────────┐   │  ← スキャンするとここに JAN が入る
│  │ 4901234567890                │   │
│  └─────────────────────────────┘   │
│                                     │
│  商品名（自動表示）                  │
│  おにぎり 梅                       │
│                                     │
│  箱入数  [  24  ]  箱数 [  0  ]     │  ← タップして入力
│  端数    [  5   ]                   │
│  数量（自動） 5                     │
│                                     │
│  ロケーション  [オフィス ▼]         │  ← タップで選択
│                                     │
│  ┌─────────────────────────────┐   │
│  │        保 存                 │   │  ← タップで保存
│  └─────────────────────────────┘   │
│                                     │
│  ※保存後 → JAN欄をクリアし、       │
│    再び JAN にフォーカス（次の      │
│    スキャン待ち）                   │
├─────────────────────────────────────┤
│ [出荷][移動][入荷][棚卸][差異][商品] │  ← 固定メニュー
└─────────────────────────────────────┘
```

**操作の流れ（想定）**

1. 画面を開く → JAN の入力欄にフォーカスが当たっている。
2. バーコードをスキャン → JAN が入力され、商品名・箱入数などが自動表示される（マスタ参照）。
3. 必要なら 箱数・端数・ロケーション をタップして入力・変更。
4. 「保存」をタップ → 1 件登録。画面上は「登録しました」と短く表示し、JAN 欄を空にして再び JAN にフォーカス。
5. そのまま次の商品をスキャンして 2～4 を繰り返す。

## 4. 出荷（連続スキャン）の例

出荷は「1 点ずつスキャンして登録」なので、**JAN だけ入力してすぐ保存** する流れにできます。

```
┌─────────────────────────────────────┐
│  出荷                               │
├─────────────────────────────────────┤
│                                     │
│  [JANコード]  ★常にここにフォーカス  │
│  ┌─────────────────────────────┐   │
│  │ 4901234567890                │   │  ← スキャン → 自動保存 or
│  └─────────────────────────────┘   │     「登録」タップ
│                                     │
│  商品名（自動） おにぎり 梅          │
│  出荷元 [本社 ▼] （初期値で省略可）  │
│                                     │
│  ┌─────────────────────────────┐   │
│  │    登録（1点）               │   │
│  └─────────────────────────────┘   │
│                                     │
│  ※登録後 → JAN クリア＋フォーカス   │
│    戻し。次のスキャンへ。           │
└─────────────────────────────────────┘
```

- オプション: 「JAN が入ったら自動で保存する」にすると、スキャン → 即保存 → フォーカス戻し、の流れで連続スキャンが可能です。

## 5. イメージ確認用のデモ HTML

「スキャン → タップで次へ → 保存 → 次のスキャン」の流れを、**実際にスマホで触ってイメージできる** ように、静的 HTML のデモを用意しました。

- **ファイル**: `docs/inventory-app-ui-demo.html`
- **開き方**: スマホのブラウザでこのファイルを開く（Google ドライブにアップロードして「プレビュー」でも可）。GAS は不要です。
- **できること**:
  - 最初に JAN 欄にフォーカスが当たる。
  - JAN を手入力またはスキャンで入力すると、ダミーの「商品名」が表示される。
  - 数量・ロケーションをタップして入力し、「保存」で 1 件登録（画面内でリスト表示のみ）。
  - 保存後、JAN をクリアして再び JAN にフォーカスし、次のスキャンに備える。

このデモで「タップしながら進める」感覚を確認したうえで、GAS 実装時に同じレイアウト・フォーカス制御を採用できます。
