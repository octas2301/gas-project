# 消費税輸出免除不適格用連絡一覧表 自動作成プロジェクト 要件定義書

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| **プロジェクト名** | 消費税輸出免除不適格用連絡一覧表の自動作成 |
| **目的** | 消費税還付申請のための書類整備 |
| **対象プラットフォーム** | Shopee Thailand（越境EC） |
| **実装環境** | Google Apps Script（GAS）、Google スプレッドシート、Google ドライブ |

---

## 2. 業務フロー概要

```
[人間] ①特定フォルダに輸出許可書PDFを保存
[人間] ②空のGoogleスプレッドシートを作成
[GAS]  ③転記済みフォルダの作成
[GAS]  ④スプレッドシートに列項目を設定
[GAS]  ⑤PDFから項目を読み取り転記
[GAS]  ⑥複数AIで読み取り検証（正解率向上）
[GAS]  ⑦転記完了PDFを転記済みフォルダへ移動
```

---

## 3. 手順の詳細要件

### 3.1 人間が行う作業

| 手順 | 作業内容 | 備考 |
|------|----------|------|
| ① | **特定フォルダに輸出許可書のPDFを保存** | 監視対象フォルダ。未処理PDFのみが対象。 |
| ② | **空のGoogleスプレッドシートを作成** | 転記先。フォルダIDまたはスプレッドシートIDをGAS側で参照する設定が必要。 |

### 3.2 プログラミング（GAS）で対応する処理

| 手順 | 処理内容 | 詳細 |
|------|----------|------|
| ③ | **転記済みフォルダの作成** | 特定フォルダ直下に「転記済み」等の名前でフォルダを作成。存在しない場合のみ作成。 |
| ④ | **スプレッドシートの列項目設定** | 下記「4. スプレッドシート仕様」を参照。 |
| ⑤ | **PDFからの読み取りと転記** | 下記「5. PDF読み取り仕様」を参照。 |
| ⑥ | **複数AIによる読み取り検証** | 下記「6. 読み取り検証仕様」を参照。 |
| ⑦ | **転記完了PDFの移動** | 処理完了したPDFを転記済みフォルダに移動。 |

## 4. スプレッドシート仕様

### 4.1 列項目（ヘッダー行）

| 列番号 | 列名（ヘッダー） | 説明 |
|--------|------------------|------|
| A | No. | 連番（1から採番） |
| B | 海　外　客　先 | 輸出先の顧客名（仕向人） |
| C | 取引年月日 | 申告年月日（日付形式） |
| D | 輸出金額 | 申告価格（金額・数値） |
| E | Invoice No. | 申告番号 |

### 4.2 データ形式

- **No.**：整数（1, 2, 3 …）
- **海　外　客　先**：文字列（例：SR TOYSCAFE SHOP）
- **取引年月日**：日付（YYYY/MM/DD または スプレッドシートの日付型）
- **輸出金額**：数値（カンマ・通貨記号は除去して格納するか、表示用にフォーマットするか要検討）
- **Invoice No.**：文字列（スペース含む可。例：180 0278 4060）

## 5. PDF読み取り仕様

### 5.1 読み取り項目と転記先の対応

| PDF上の項目（ラベル） | 転記先列 | 備考 |
|----------------------|----------|------|
| 仕向人 | 海　外　客　先 | 例：「仕向人 - SR TOYSCAFE SHOP」→ 「SR TOYSCAFE SHOP」を抽出 |
| 申告年月日 | 取引年月日 | 形式：YYYY/MM/DD（例：2025/07/09） |
| 申告価格 | 輸出金額 | 例：「¥4,579」→ 数値 4579 または 「4,579」を格納 |
| 申告番号 | Invoice No. | スペース区切り可（例：180 0278 4060） |

### 5.2 PDF上の表示例（参考）

- **仕向人**：ラベル「仕向人」の右側、ハイフン等の区切り後に「SR TOYSCAFE SHOP」のような文字列。
- **申告年月日**：ラベル「申告年月日」の下に「2025/07/09」形式。
- **申告価格**：ラベル「申告価格」の右に「¥4,579」等。カンマ・¥は除去して数値として扱う想定。
- **申告番号**：ラベル「申告番号」の下に「180 0278 4060」のようなスペース入り番号。

### 5.3 技術的な注意点

- GASからPDFのテキスト抽出には **DriveApp** でBlob取得後、**Utilities** や外部API、または **Document App** では直接不可なため、**PDFをテキスト化する手段**（例：外部サービス、Base64＋Cloud Vision API、または他アプリ連携）の選定が必要。
- 画像PDFの場合はOCR（例：Google Cloud Vision API）が必要。

---

## 6. 読み取り検証仕様（複数AIチェック）

### 6.1 目的

- OCR/テキスト抽出の誤りを減らし、転記の正解率を上げる。

### 6.2 方針（案）

- 同一項目を **複数のAI（または複数回の解釈）** で読み取り結果を比較。
- 一致すればその値を採用、不一致の場合は：
  - 多数決で採用、または
  - 要確認フラグを立てて人間確認行に残す、または
  - 信頼度の高い方を採用するルールを定義。
- 利用する「AI」の候補：
  - 同一APIの複数回呼び出し（temperature等を変える）、
  - 複数プロバイダ（例：OpenAI / Claude / Gemini 等）の併用、
  - またはGASから呼び出せるテキスト検証用API。

### 6.3 検証対象

- 仕向人（海　外　客　先）
- 申告年月日（取引年月日）
- 申告価格（輸出金額）
- 申告番号（Invoice No.）

## 7. 挙動・制約要件

### 7.1 GAS 実行時間制限（6分の壁）

| 要件 | 内容 |
|------|------|
| 制限対策 | 1回の実行は **5分30秒で自ら停止** する。 |
| 継続処理 | 停止時に **「1分後に続きから処理を実行する」** 指示を自ら作成する。 |
| 実装案 | トリガーで「1分後に自分を再実行する」スクリプトを起動する、またはメモ（プロパティ等）に「次に処理するファイルID・行番号」を保存し、次回実行時に続きから再開。 |

### 7.2 自動起動スケジュール

| 項目 | 内容 |
|------|------|
| 実行時間帯 | **東京時間（JST）の AM1:00 ～ AM7:00 の間** |
| 動作 | この時間帯に自動起動し、特定フォルダを確認。未処理のPDFがあれば処理を開始。 |
| 繰り返し | 指定時間内は **読み取り・転記を繰り返し** 実行（5分30秒停止→1分後に再開を繰り返す想定）。 |

### 7.3 フォルダ・ファイル

- **特定フォルダ**：出力許可書PDFを格納するGoogleドライブのフォルダ。フォルダIDをスクリプトプロパティ等で保持。
- **転記済みフォルダ**：同一の「特定フォルダ」直下に作成。転記完了したPDFのみ移動対象。
- **転記失敗フォルダ**：同一の「特定フォルダ」直下に作成。転記失敗・壊れたPDF・パスワード付きPDFを格納。
- **対象ファイル**：未処理のPDF（転記済み・転記失敗フォルダに移動済みのものは対象外）。

### 7.4 スプレッドシートの参照方法

- 転記先スプレッドシートは **スプレッドシートID** をプロパティまたは設定シートで指定する想定。
- 同一スプレッドシートに複数日・複数PDFの結果を追記する形とする。

#### 質問意図が不明だった項目の補足（D4・E3・F2）

- **D4（APIキーの管理）**  
  Gemini・OpenAI 等の **APIキーをどこに保存するか** という意味です。  
  例：GASのスクリプトプロパティに保存する／設定シートに記載して読み込む／どちらでもよい。  
  → ご希望が決まり次第、回答欄に追記してください。

- **E3（継続処理の記憶）**  
  5分30秒で処理を止めたあと、**「次にどのPDFから再開するか」などの情報**を、1分後の再実行時に読み出して続きから処理する必要があります。  
  その情報を **GASの ScriptProperties（スクリプトの保存領域）に保存する** 運用でよいか、という意味です。
