# 楽天・Yahoo! 同時出品 — 運用構成メモ

最終更新: 2026-02-21

---

## 1. 開発しやすい構成（推奨）

**Contact をオーナーにし、CSV保存フォルダだけ Octas2301 のドライブに置く**構成が、開発・運用とも扱いやすい。

| 項目 | 推奨 |
|------|------|
| **スプレッドシート・GAS のオーナー** | contact@octas2301.com |
| **CSV保存フォルダの場所** | octas2301@gmail.com の Google ドライブ（従来どおり） |
| **フォルダの共有** | Octas2301 が当該フォルダを **contact@octas2301.com に「編集者」で共有** する |

**こうするとどうなるか**

- Contact がオーナーなので、**トリガー自動作成が常に成功**する（一括出品「実行」で 1 分後トリガーが自動作成される）。
- API キー・Make がもともと Contact なので、**開発・運用が 1 アカウントに集約**される。
- CSV はこれまでどおり **Octas2301 のドライブ内のフォルダ** に保存される。  
  GAS は「トリガー実行時＝オーナー（Contact）」として動くため、**そのフォルダを Contact が編集できる必要がある**。  
  → Octas2301 がフォルダを contact@octas2301.com に「編集者」で共有すれば、Contact オーナーでも保存・アーカイブは問題ない。

**譲渡手順の要点**（※同じドメインでないと譲渡不可 — 下記「コピーで新規作成」を参照）

1. スプレッドシートの「共有」→ contact@octas2301.com を **オーナーに昇格**（または「オーナー権限を譲渡」）。
2. 必要なら GAS プロジェクトも Contact がオーナーのスプレッドシートに紐づいていることを確認。
3. Octas2301 のドライブで **CSV 保存用・アーカイブ用フォルダ** を contact@octas2301.com に **編集者** で共有。
4. Contact で GAS を一度実行し、表示される権限（トリガー追加含む）を承認。

**※オーナー譲渡ができない理由**  
Google の仕様で、オーナー権限は「**現在のオーナーと同じドメイン内**のユーザーにのみ」譲渡できます。  
`octas2301@gmail.com`（gmail.com）と `contact@octas2301.com`（octas2301.com）はドメインが異なるため、譲渡できません。  
→ **代替案: コピーして新規ファイルにする**（下記セクション 1.1）。

---

### 1.1 コピーして新規ファイルにする場合（Contact を実質オーナーにする）

**手順の流れ**

1. **contact@octas2301.com** でログインし、元のスプレッドシートを開く（編集者で開ける）。
2. **ファイル → コピーを作成** で新しいスプレッドシートを作成する。  
   → **コピーしたファイルのオーナーは「コピーを作成した人」= Contact** になる。  
   → 紐づく GAS もコピーされ、**新しい GAS プロジェクトのオーナーも Contact** になる。
3. 以降は「新しいスプレッドシート」を本番として使う。

**こうするとどうなるか**

| 項目 | 結果 |
|------|------|
| オーナー | contact@octas2301.com（コピー作成者） |
| シート・GAS の内容 | 元と同じ（コード・シート構成はコピーされる） |
| トリガー自動作成 | ✅ Contact がオーナーなので成功する |
| CSV 保存フォルダ | コード内のフォルダ ID はそのまま → **Octas2301 のドライブの同じフォルダ**を使う。**そのフォルダを Contact に「編集者」で共有**すれば保存できる。 |

**コピー後に必ずやること（チェックリスト）**

| # | 作業 | 備考 |
|---|------|------|
| 1 | **Script Properties を再設定** | コピーした GAS プロジェクトには**プロパティは引き継がれない**。GAS エディタ → プロジェクトの設定 → スクリプト プロパティで以下を登録する。 |
|   | `OPENAI_API_KEY` | 未使用なら省略可 |
|   | `GEMINI_API_KEY` | 同上 |
|   | `RAKUTEN_LICENSE_KEY` | 楽天 RMS 用 |
|   | `RAKUTEN_SERVICE_SECRET` | 楽天 RMS 用 |
|   | `NOTIFICATION_EMAIL` | 完了メール送信先 |
|   | `RAKUTEN_DELETE_WEBAPP_URL` | 楽天削除用 Web アプリの URL（下記 #3 のデプロイ後に設定） |
| 2 | **トリガーを登録** | コピーしたプロジェクトにはトリガーはコピーされない。必要に応じて「runBatchExportFromTrigger」1 分ごと、「runReflectionCheck」5 分ごとなどを手動で追加。 |
| 3 | **Web アプリをデプロイ** | 楽天削除用 URL を使う場合、新しい GAS で「デプロイ → 新しいデプロイ」→ Web アプリをデプロイし、発行された URL を `RAKUTEN_DELETE_WEBAPP_URL` に設定。 |
| 4 | **CSV 保存フォルダの共有** | Octas2301 のドライブ内の CSV 保存用・アーカイブ用フォルダを **contact@octas2301.com に「編集者」で共有**。 |
| 5 | **Make・外部リンクの更新** | Make やブックマークで「元のスプレッドシートの URL」を参照している場合は、**新しいスプレッドシートの URL / ファイル ID** に差し替える。 |

**まとめ**

- **コピーして新規作成**すれば、オーナー譲渡なしで **Contact がオーナー**の「新しい本番」ができる。
- フォルダの**保存場所は Octas2301 のドライブのまま**でよく、フォルダ ID はコード内定数のためそのままでよい。共有だけ Contact に付与する。
- 手間は「Script Properties・トリガー・Web アプリ・リンクの付け直し」だが、一度やっておけば以降は現状（Octas2301 オーナー）より運用しやすい。

---

### 1.2 逆に Octas2301 に全ての権限を付ける場合

**やることのイメージ**

- スプレッドシート・GAS のオーナーは**すでに Octas2301** なので、ファイルのコピーや譲渡は不要。
- 「全ての権限」= **API キー・Make を Octas2301 アカウントで持つ**ようにする、という意味にすると：
  - **楽天 RMS**（ライセンスキー・シークレット）を **Octas2301 の楽天アカウント**で取得し直し、Script Properties に設定。
  - **Yahoo! API**（クライアントID・シークレット等）を **Octas2301 の Yahoo アカウント**で取得し直し、設定シートまたは Script Properties に設定。
  - **Make** を **Octas2301 のアカウント**で利用するようにする（既存シナリオを Octas2301 で作り直すか、接続を Octas2301 に差し替え）。
  - 通知メール・削除用 Web アプリ URL などは、Octas2301 の GAS の Script Properties にそのまま設定すればよい。

**手間のポイント**

- 各サービスの **API キーは「そのアカウントに紐づく」** ため、Octas2301 で使うには **Octas2301 用に取得し直す**必要がある（既存の Contact 用キーを「写すだけ」では、アカウント違いで制限や不具合が出る場合がある）。
- 楽天・Yahoo などはアカウント単位の認証なので、**別アカウント用のキーを流用し続けると規約や運用上よくなない**こともある。その意味でも「Octas2301 に全て寄せる」なら、**各サービスで Octas2301 用のキーを新規取得**する手間が発生する。

---

### 1.3 どちらが手間が少ないか（比較）

| 比較項目 | Contact オーナー（コピーで新規） | Octas2301 に全ての権限 |
|----------|----------------------------------|-------------------------|
| **ファイル・GAS** | コピー 1 回（新しい本番ができる） | そのまま（変更なし） |
| **Script Properties** | 既存の値を**写すだけ**（Contact が持っているキーをそのまま使える） | Octas2301 用に**キーを取得し直す**必要あり（楽天・Yahoo 等） |
| **トリガー** | 新 GAS に 2 本程度を手動登録 | 現状のまま（オーナーが設定済みなら不要） |
| **Web アプリ** | 新 GAS で 1 回デプロイ + URL 設定 | そのまま |
| **フォルダ** | Octas2301 が Contact に共有するだけ | そのまま（オーナーのドライブのまま） |
| **Make** | 新しいスプレッドシート URL に差し替え | **Octas2301 アカウントで Make をやり直す**か、接続を Octas2301 に変更 |
| **API キー・認証** | **取得し直し不要**（既存の Contact 用を写すだけ） | **楽天・Yahoo 等を Octas2301 で取得し直し** |

**結論：手間が少ないのは「Contact オーナー（コピーで新規）」**

- **Contact オーナー**：キー・Make は Contact のまま使えるので「写す・登録し直す」だけで済む。**新規でキーを取り直す作業がほぼない**。
- **Octas2301 に全て**：トリガーやファイルは触らなくてよいが、**各サービスで Octas2301 用の API キー・認証の取得し直し**と、**Make を Octas2301 で作り直す（または接続変更）** が発生し、手間と確認作業が増える。

運用を **1 アカウントに寄せたい**だけであれば、**既に API キー・Make を持っている Contact をオーナーにする（コピー）方が、やることは少なくて済みます**。

**開発環境（Cursor / AI 連携）について**

- **登録アカウントが contact@octas2301.com の場合**、GAS・スプレッドシートのオーナーも Contact にしておくと連携しやすいです。
- 理由：コード編集（Cursor）・clasp push・GAS の実行・トリガー設定・Script Properties の編集が **すべて同じ Contact アカウント**で完結するため、権限の食い違いが起きにくく、「AI が提案した手順をそのまま実行できる」ことが多くなります。
- 逆にオーナーが Octas2301 のままだと、Contact で編集した内容を反映したりトリガーを付けたりする際に、いったん Octas2301 で操作する必要が出ることがあります。

### 2.2 Contact にオーナー譲渡した場合

| 役割 | アカウント | 備考 |
|------|------------|------|
| **オーナー** | contact@octas2301.com | トリガー自動作成・手動登録が可能。開発・運用を 1 アカウントで完結できる。 |
| **編集者** | octas2301@gmail.com | シート編集・メニュー実行は従来どおり。CSV フォルダは「自分のドライブ内のまま」で、Contact に共有するだけ。 |
| **編集者** | 3358miho@gmail.com | 同上。 |

**変わること**

- 一括出品「実行」を **誰が押しても**、Contact がオーナーなのでトリガー自動作成が成功する（オーナー登録の「1 分ごと」トリガーは不要）。
- GAS の「実行元」はオーナー（Contact）になるため、**CSV 保存先フォルダは「Contact が編集できるフォルダ」である必要がある** → Octas2301 のフォルダを Contact に編集者共有すればよい。

**変わらないこと**

- CSV の**保存場所**は Octas2301 のドライブのまま（フォルダ ID は設定シートや Script Properties のまま）。
- Make・API キーは従来どおり contact@octas2301.com で問題なし。

## 4. 実行者による制約（GAS）

- **トリガー作成**は「**プロジェクトのオーナー**」でないとできない（編集者では権限不足）。
- オーナーが **Octas2301** のとき：オーナーで実行すればトリガー自動作成可能。編集者で実行した場合は「1 分ごと」をオーナーが手動で 1 本登録する必要がある。
- オーナーが **Contact** のとき：実行者が誰でも、トリガーは Contact 権限で動くため自動作成が成功する。開発しやすい。

| 操作 | 現状（Octas2301 オーナー） | Contact オーナー後 |
|------|----------------------------|--------------------|
| トリガー自動作成（一括出品） | オーナー実行時のみ ✅ | 誰が実行しても ✅ |
| 手動トリガー登録 | オーナーのみ | Contact のみ |
| CSV 保存 | オーナーのドライブ or 共有フォルダ | **Octas2301 のフォルダを Contact に編集者共有** すればそのまま利用可能 |

## 5. 一括出品の運用フロー（整理）

### 現状（Octas2301 オーナー）

- **編集者が実行**: 予約のみ保存。オーナーが「runBatchExportFromTrigger」1 分ごとを 1 本登録しておく必要あり。
- **オーナーが実行**: 予約＋トリガー自動作成（スコープ承認済みなら）。

### Contact オーナー後

- **誰が実行しても**: 予約＋トリガー自動作成。1 分後に実行され、終了後は自動作成トリガーのみ削除。

## 6. 関連ファイル

- 一括出品: `コード.js` — `scheduleBatchExport`, `runBatchExportFromTrigger`, `deleteOwnBatchExportTrigger`
- トリガー用スコープ: `appsscript.json` — `oauthScopes` に `script.scriptapp` を追加済み
- CSV 保存先: 設定シートまたは Script Properties の `CSV_SAVE_FOLDER_ID` / `CSV_ARCHIVE_FOLDER_ID`（コード.js で `DriveApp.getFolderById` に渡している）
