# 新 Agent 用 引き継ぎ（同じフォルダ前提）

**前提**: 新しい Agent も同じ **gas-project** フォルダを開いている。このファイルを「見てね」と指定すれば、どこに何が書いてあるかが分かり、開発を引き継げる。

---

## チャットで伝える一言（コピペ用）

新しい Agent の**最初のメッセージ**で、次のどちらかを送れば十分です。

```
新Agent用_引き継ぎ文.md を読んで、そこに書いてある「読むべきファイル」を順に参照したうえで、商品情報取得とマスタ効率化の開発を引き継いで続けてください。
```

または、もう少し短く:

```
新Agent用_引き継ぎ文.md を見て、引き継ぎのとおりに開発を続けてください。
```

## このフォルダの主なファイルと目的

| ファイル | 目的・役割 |
|----------|-------------|
| **HANDOVER.md** | 開発引継ぎの**本体**。前提ルール・Yahoo/楽天の確定仕様・シート構成・解決済み課題・残タスク・**新プロジェクト「商品情報取得とマスタ効率化」の要件定義（§10）**・参照すべき行番号一覧（§11）まで、すべてここにまとまっている。 |
| **コード.js** | メニュー(onOpen)・一括出品実行・楽天CSV出力・AI情報取得・マスタ同期(syncAiDataToMaster)・Webアプリ削除APIなど、**「つなぎ」と楽天まわり**。楽天の既存ロジックは変更禁止（聖域）。 |
| **Yahoo.js** | **Yahoo! 専用**。マスタ＋マッピングから商品データを組み立て、画像API・editItem・setStock・submitItem を実行。認証リトライ・定期購入価格・発送日情報・グルーピングの改修もここに反映済み。 |
| **新Agent用_引き継ぎ文.md** | このファイル。**引き継ぎの入口**。「どのファイルが何のためか」「どのファイルを読んで続けるか」をまとめた目次。 |
| **運用構成メモ.md** | 運用上のメモ（必要に応じて参照）。 |
| **.clasp.json** | clasp 用の scriptId。GAS への push/pull に使用。 |

その他、`check-rakuten-32byte.js` や `export-tax-list/` 内のドキュメントなどは、該当機能の開発時に参照。

## 開発を引き継ぐときに読むべきファイルと順序

1. **このファイル（新Agent用_引き継ぎ文.md）**  
   → 全体の地図と「次に何を読むか」を把握する。

2. **HANDOVER.md**  
   - **§0〜§2**: 前提（楽天聖域・Yahoo分離）、現状、Yahoo 確定仕様。  
   - **§3〜§5**: マッピング制御、解決済み課題、EC出品の要件。  
   - **§8〜§9**: 残タスク・要件棚卸し。  
   - **§10**: **新プロジェクト「商品情報取得とマスタ効率化」の要件・ファイル分割案・調査項目CSV 方針**。ここを読まずに新プロジェクトに手を付けないこと。  
   - **§11**: 参照すべきファイル・行番号一覧。実装を触るときの入口。

3. **コード.js** または **Yahoo.js**  
   → これから変更・追加する機能に応じて、HANDOVER.md §11 の行番号を頼りに該当箇所を読む。

**重要**: 詳細な仕様・経緯・改修内容は **HANDOVER.md と各 .js 内**にあり、このファイルには「どのファイルをどう読んで続けるか」だけを書いている。

---

## 絶対に守ること（1行要約）

- **楽天の既存ロジック（generateRakutenCSV 等）は変更・干渉しない。Yahoo は楽天と分離して管理する。**  
  → くわしくは HANDOVER.md §0。

---

## 新プロジェクトについて

- **商品情報取得**: 仕入元・セラセン・既存CSV等から取得し、AI情報取得data やマスタに流し込む。  
- **マスタ効率化**: マスタ列と取得項目の対応をはっきりさせ、送料・手数料等は商品出品情報CSVを参照して補完・計算する。  

**要件・ファイル分割案・調査項目CSV の考え方**はすべて **HANDOVER.md §10** に書いてある。§10 を読んだうえで、既存の コード.js・Yahoo.js を壊さない範囲で設計・実装すること。

## まとめ
