### 手順1.5：追加機能の設定（任意）

- **複数AI検証**：設定シートに A列 `複数AI検証回数`、B列 `2` または `3` を追加すると、**同じPDFを複数回並列で**読み取り、多数決で転記します（所要時間は1回分で済む）。未設定または 0/1 のときは1回だけ読み取ります。**1万件を短時間で処理する場合は 1 または未設定**にすると最も速くなります。
- **時間驅動トリガー**：Apps Script で関数 **`createScheduleTrigger`** を1回実行すると、「1分ごとに runScheduled を実行」するトリガーが1つ登録されます。runScheduled は **設定シートで指定した最大5枠の時間帯のいずれかの間だけ** 実際に処理し、それ以外は何もしません。手動で `run` を実行する場合は時間制限なしで動きます。  
  **時間帯の設定**：設定シートに次の項目を追加します（時間は0〜23の整数。例: 1＝1時、7＝7時）。未設定の枠は無視され、1つも設定がない場合はデフォルトで 1時〜7時 になります。
  | A列（設定項目） | B列（値） |
  |-----------------|-----------|
  | 実行時間帯1開始 | 1 |
  | 実行時間帯1終了 | 7 |
  | 実行時間帯2開始 | 10 |
  | 実行時間帯2終了 | 12 |
  | 実行時間帯3開始 | 15 |
  | 実行時間帯3終了 | 18 |
  | 実行時間帯4開始 | （空でよい） |
  | 実行時間帯4終了 | （空でよい） |
  | 実行時間帯5開始 | （空でよい） |
  | 実行時間帯5終了 | （空でよい） |
- **ログ**：実行のたびに「ログ」シートに 実行日時・結果・処理PDF数・メッセージ が1行追記されます。シートがなければ自動作成されます。
- **メール通知**：設定の「通知メールアドレス」にメールアドレスを入れておくと、実行結果（成功・対象なし・エラー・途中終了）がそのアドレスに送信されます。初回はメール送信の権限承認が必要な場合があります。

### 手順2：転記用スプレッドシートの「設定」シートを確認する

1. **転記用スプレッドシート**（このGASが紐づいているスプレッドシート）を開く。
2. **「設定」** シートを開く。
3. 次の項目が **A列に項目名、B列に値** で入っているか確認する。足りない行は追加する。

| A列（設定項目） | B列（値） |
|-----------------|-----------|
| 監視フォルダID | （PDFを入れるフォルダのDriveフォルダID） |
| 転記先スプレッドシートID | （このスプレッドシートのIDでよい） |
| 転記済みフォルダ名 | 転記済み |
| 転記失敗フォルダ名 | 転記失敗 |
| 最大実行時間秒 | 240（未設定時。6分制限前にメール・トリガー登録のため90秒余裕で停止） |
| 通知メールアドレス | （結果を受け取りたいメールアドレス） |
| 処理対象年 | 2025 |
| **GeminiAPIキー** | **（Google AI Studioで取得したAPIキー）** |

- **監視フォルダID**：テスト用なら「輸出許可書_テスト」など、PDFを1〜2個だけ入れたフォルダのIDを入れる。本番なら本番フォルダのID。
- **GeminiAPIキー**：https://aistudio.google.com/apikey で「Create API key」から発行し、B列に貼る。

### 手順3：「2025年」シートとヘッダーを用意する

1. 同じスプレッドシートに **「2025年」** という名前のシートがなければ、手動でシートを追加し名前を「2025年」にする。
2. あるいは Apps Script で **関数 `testSheetSetup`** を実行すると、「〇〇年」シートとヘッダーが自動作成される（処理対象年が設定されていればその年になる。設定がなければ今年の年になる）。
3. 「2025年」シートの **1行目** に次のヘッダーがあるか確認する。ない場合は手動で入力するか、`testSheetSetup` を実行する。
   - `No.` | `海　外　客　先` | `取引年月日` | `輸出金額` | `Invoice No.` | `要確認` | `元ファイル名`

### 手順4：Step 6 の動作確認（PDF読み取り・転記・移動）

1. **監視フォルダ**（設定の「監視フォルダID」のフォルダ）に、**輸出許可書のサンプルPDFを1つ** 入れる。  
   - テスト用フォルダを使う場合は、そこに1つだけ入れる。

2. **Apps Script を開く**  
   - 転記用スプレッドシート → **拡張機能** → **Apps Script**

3. **`run` を実行する**  
   - 上部「実行する関数」で **`run`** を選択 → **▶ 実行**

4. **確認する**
   - **実行ログ**（表示 → ログ）：エラーが出ていないか。「未処理PDF: 1 件」「今回の実行で全件処理しました。」などが出ていればOK。
   - **「2025年」シート**：2行目以降に、No.・海　外　客　先・取引年月日・輸出金額・Invoice No.・要確認・元ファイル名 が1行以上追加されているか。
   - **監視フォルダ**：対象のPDFがなくなっているか。
   - **「転記済み」フォルダ**：そのPDFが移動しているか。

5. **エラーになった場合**
   - PDF は **「転記失敗」** フォルダに移動している想定。実行ログのエラー内容を確認する。
   - 「Gemini API エラー」→ APIキー・課金設定を確認。
   - 「設定シートに〜がありません」→ 手順2の設定項目を再確認。

## 補足：テスト用フォルダを使う場合

本番の監視フォルダに1万件以上ある場合は、**別の「テスト用」フォルダ**を用意し、そのフォルダIDを一時的に「監視フォルダID」に入れて動作確認することを推奨する。確認後、本番のフォルダIDに戻す。
