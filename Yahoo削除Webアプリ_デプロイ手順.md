# Yahoo!商品削除 Webアプリ デプロイ手順

## 前提

- 対象: Yahoo.js の `doGet` / `doPost`（削除確認ページ・削除実行）
- スプレッドシートに紐づいたGASプロジェクトからデプロイします

---

## Step 1: コードをGASに反映

ローカルで修正した場合は、以下で反映してください。

```bash
clasp push
```

---

## Step 2: デプロイの作成

1. **スプレッドシート**を開く
2. メニュー **拡張機能** → **Apps Script** でエディタを開く
3. 右上 **「デプロイ」** → **「新しいデプロイ」**
4. 種類で **「ウェブアプリ」** を選択
5. 設定:
   - **説明**: 例）Yahoo削除確認 v1
   - **次のユーザーとして実行**: **自分**
   - **アクセスできるユーザー**: **全員**（削除リンクをメールで開くため）
6. **「デプロイ」** をクリック
7. **ウェブアプリのURL** が表示されるので **コピー**（例: `https://script.google.com/macros/s/xxxxx/exec`）

---

## Step 3: 削除フローの確認

### 3.1 確認ページを開く

次のURLをブラウザで開きます（`item_code` と `name` は例です）。

```
https://script.google.com/macros/s/【あなたのデプロイID】/exec?action=delete&item_code=test-item-001&name=テスト商品
```

- デプロイ後にコピーしたURLの末尾が `/exec` なので、その前に `?action=delete&item_code=...&name=...` を付ける
- 「Yahoo!商品削除確認」のページが表示されればOK

### 3.2 削除実行（テスト時）

- **「削除する」** を押すと `doPost` が呼ばれ、Yahoo! APIで削除が実行されます
- テスト時は実際に削除されたくない商品のコードは使わないでください
- **「キャンセル」** でウィンドウを閉じるだけならAPIは呼ばれません

---

## Step 4: メール内リンクの形式

削除用メールに記載するリンクは次の形式にします。

```
【デプロイURL】?action=delete&item_code=【商品コード】&name=【商品名（URLエンコード）】
```

- 商品名に `&` や `=` が含まれる場合は **URLエンコード**（`encodeURIComponent`）すること
- スプレッドシートの「削除用リンクをメールで送る」機能でこのURLを生成している場合は、その処理でデプロイURLを使うようにしてください

---

## 再デプロイ（コード変更後）

1. Apps Script エディタで **「デプロイ」** → **「デプロイを管理」**
2. 該当のウェブアプリの **鉛筆アイコン（編集）** をクリック
3. **「バージョン」** を **「新バージョン」** に変更
4. **「デプロイを更新」**

※URLは変わらないので、メールのリンク形式を変える必要はありません。

---

## トラブルシューティング

| 現象 | 対処 |
|------|------|
| 「無効なリクエストです」と表示される | URLに `action=delete` と `item_code=...` が含まれているか確認 |
| 削除するを押しても反応しない／エラー | ブラウザの開発者ツールでネットワークエラーを確認。doPost の実行ログを「実行数」で確認 |
| 認証画面が繰り返し出る | 別ブラウザやシークレットで「全員」でデプロイしたURLを一度開き、表示されるGoogleアカウントで許可 |
| マスタの削除フラグが更新されない | `deleteYahooItem` 内の `updateMasterDeleteFlag` とマスタシートの列名・列位置を確認 |
